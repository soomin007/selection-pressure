# UI 핸드오프 문서 — 적자생존 (Selection Pressure)

> 이 문서 하나로 **코드를 안 보고도** 이 게임의 맥락과 UI가 붙는 방식을 파악하도록 쓴다.
> 외부에서 UI를 새로 짜거나 다듬을 사람이 읽는 대상. (2026-07-06 기준. 코드가 바뀌면 이 문서도 갱신)

---

## 0. 한 줄 요약

한 종(種)을 길러 생태계 정점에 올리는 **관전형 진화 로그라이크**. 플레이어는 직접 조작하지 않고,
라운드 사이에 **카드(형질)를 고르기만** 한다. 규칙기반 생태 시뮬이 그 선택의 결과를 살아 움직이게 한다.
**모바일 세로 우선**, 정적 URL(GitHub Pages) 배포. 스택: TypeScript + PixiJS v8 + Vite.

플레이어는 조종자가 아니라 **관찰자·사육사**다. UI 톤은 "차분한 자연 관찰 다큐/야외 도감"(뒤 §10).

## 1. 가장 중요한 구조 원칙 — 로직과 그림의 분리

코드는 네 층으로 갈린다. UI를 새로 짜도 이 경계는 지켜야 한다.

| 디렉터리 | 역할 | 규칙 |
|---|---|---|
| `src/sim/` | 순수 시뮬레이션(월드·개체·먹이·행동·게놈·보스·대멸종) | **PixiJS import 금지.** 화면 없이 돌고, 단위 테스트 가능하고, **결정론**(같은 시드 → 같은 결과). `Math.random()` 금지 — 시드 RNG만. |
| `src/game/` | 런 구조·상태기계(`Game` 클래스), 카드·드래프트, 메타 진행(레벨·언락) | 순수 TS. sim을 구동하고 UI에 콜백/메서드로 상태를 노출. |
| `src/render/` | PixiJS로 월드를 **그리기만**(생물 스프라이트·지형·HUD 캔버스) | sim 상태를 **읽어서** 그린다. sim을 바꾸지 않는다. |
| `src/ui/` | 캔버스 위 **HTML 오버레이** 패널(로비·카드·결과·정보 카드 등) | `Game`의 콜백/메서드로만 붙는다. DOM + CSS. |

**핵심:** UI(HTML)와 렌더(Pixi 캔버스)는 서로 다른 두 레이어다. 게임 로직(`Game`)이 둘 사이의 유일한
진실이다. **UI를 갈아엎어도 `Game`의 인터페이스(§4)만 그대로 지키면 sim/render는 안 건드려도 된다.**

## 2. 화면 구성(한 판 동안 무엇이 보이나)

세로 화면(모바일 기준) 위에 이런 요소가 겹쳐 있다. 모두 캔버스 위 오버레이(HTML) 또는 캔버스 내부(Pixi).

```
┌─────────────────────────────────────┐
│ [정보 카드: 종/야생 수, 단계, Lv바]   [1x][⏸]  │  ← hudPanel(HTML 오버레이) + 컨트롤 바(HTML, 우상단)
│                        [선택한 형질 ▸] │  ← 빌드 패널(HTML, 우측, 접이식)
│  ──진행 타임라인 막대(보스/멸종 마커)──   │  ← hudPanel(HTML 오버레이)
│                                     │
│           (월드 = 생물들이 사는 곳)      │  ← worldView(Pixi 캔버스): 지형·생물·먹이
│                                     │
│  [◎ 한 마리 관찰]         [미니맵]      │  ← 관찰 버튼(HTML) · 미니맵(Pixi 캔버스)
│  [개체 정보 카드]                      │  ← creatureCard(HTML, 좌하단, 개체 탭 시)
│  [＋][－]                             │  ← 줌 버튼(HTML, 좌하단)
│─────────────────────────────────────│
│  [드래프트 카드 3장 / 결과 / 로비]       │  ← 하단 오버레이(HTML): 상황에 따라 교체
└─────────────────────────────────────┘
```

전체 화면을 덮는 오버레이(로비·멈춤 메뉴·진척도 화면·순간 연출)는 상황에 따라 위를 덮는다.

## 3. 한 판(런)의 흐름 — 상태와 화면 전환

`Game.phase` 가 `"lobby" | "draft" | "watch" | "result"` 넷 중 하나다. 화면은 이 phase + 콜백으로 바뀐다.

```
로비(lobby)
  │  "시작" 탭 → game.beginRun()
  ▼
프리셋 선택(draft, isChoosingPreset=true)   ← 어떤 종으로 시작할지(캐릭터 선택 창)
  │  game.pickCard(i)
  ▼
관전(watch) ─── 먹이 먹어 경험치 차면 ──▶ 레벨업 드래프트(draft, 카드 3장)
  │                                        │ game.pickCard(i) 또는 skipDraft() 또는 reroll()
  │  ◀──────────────────────────────────────┘  (관전으로 복귀)
  │
  │  단계 계획(SCHEDULE)을 따라 위협이 흐른다:
  │  채집 → 채집 → 보스 → 채집 → 보스 → 대멸종
  │  (보스/대멸종은 "버티기": 끝까지 기준 개체 수 생존하면 통과)
  │
  ├─ 개체 0 = 멸종(패배) ──▶ 결과(result)
  ├─ 대멸종까지 통과 = 승리 ──▶ 결과(result)
  │
  ▼
결과(result)
  ├─ 승리 & 다음 시대 있음 → "다음 시대로" → game.continueToNextEra() → 더 센 위협으로 관전 재개
  └─ 패배 또는 최종 정복 → "새 혈통으로 시작" → game.beginRun()
```

**시대(era) 루프:** 대멸종까지 넘기면 "승리"지만, 마지막 시대(5)가 아니면 **게놈·레벨을 유지한 채 더 센
다음 시대**로 이어간다(brotato식). 시대 5의 대멸종까지 넘으면 "정복"(최종 승리).

**런 종료 연출 순서(멸종·정복 시):**
`순간 연출(momentOverlay: 멸종/정복 큰 글자·빛)` → `진척도 화면(levelUpScreen: 메타 경험치바·레벨업·해금)`
→ `결과 화면(resultPanel: 사망 원인 요약 + 다음 버튼)`. 중간 시대 승리(이어감)는 진척도 화면을 건너뛴다.

## 4. `Game` 클래스 — UI가 붙는 유일한 접점

UI는 `Game` 인스턴스의 **콜백을 설정**하고 **메서드를 호출**하고 **상태(속성)를 읽는다**. 이게 전부다.

### 4.1 콜백(UI가 설정 → Game이 호출)

```ts
game.onDraft = (cards: Card[], preview: string) => { ... }
// 드래프트를 띄울 때 호출. cards=고를 카드들, preview=안내/예고 문구.
// game.isChoosingPreset 이 true 면 "시작 프리셋 선택"(캐릭터 창), false 면 일반 카드 드래프트.

game.onResult = (result: "win"|"lose", summary: string, canContinue: boolean, progress: RunProgress|null) => { ... }
// 런이 끝났을 때. summary=사망 원인 등 결과 요약(여러 문단, "\n\n"로 나뉨).
// canContinue=승리라서 "다음 시대로" 가능한가. progress=메타 진척도(런이 진짜 끝났을 때만, 이어감이면 null).

game.onWorldChanged = (world: World) => { ... }
// 월드가 새로 만들어졌을 때(새 런·다음 시대). 렌더가 환경·종을 다시 그리고 카메라를 새 무리로 스냅.
```

### 4.2 메서드(UI가 호출)

| 메서드 | 언제 | 뜻 |
|---|---|---|
| `start()` | 부팅 1회 | 로비 진입 |
| `beginRun()` | "시작"/"새 혈통" | 새 런 시작(프리셋 선택부터) |
| `toLobby()` | 멈춤 메뉴 "로비로" | 런 버리고 로비 |
| `pickCard(index)` | 드래프트 카드 탭 | 그 카드 선택(프리셋/레벨업/시대보상 공용) |
| `skipDraft()` | 드래프트 "건너뛰기" | 형질 대신 새끼 몇 마리(프리셋 선택은 스킵 불가) |
| `reroll()` | 드래프트 "다시 뽑기" | 카드 새로 뽑기(메타 레벨 2부터 해금, 드래프트당 1회) |
| `continueToNextEra()` | 결과 "다음 시대로" | 성장 유지하고 더 센 시대로 |
| `update(deltaMS)` | 매 프레임(렌더 루프) | 시뮬 진행(배속·일시정지 반영). **관전 중에만 sim이 돈다.** |

### 4.3 읽는 상태(속성·getter)

| 이름 | 타입 | 뜻 |
|---|---|---|
| `phase` | `"lobby"\|"draft"\|"watch"\|"result"` | 현재 화면 상태 |
| `world` | `World` | 렌더가 그릴 월드(생물·먹이·지형·보스) |
| `genome` | `Genome` | 내 종 기준선 게놈(빌드 패널 "설계도"에 표시) |
| `draftCards` | `Card[]` | 지금 드래프트 카드들 |
| `isChoosingPreset` | `boolean` | 지금 드래프트가 시작 프리셋 선택인가 |
| `canReroll` | `boolean` | 지금 "다시 뽑기" 가능한가(해금 + 횟수 남음) |
| `preview` | `string` | 드래프트/단계 안내 문구 |
| `stageLabel` | `string` | 현재 단계 라벨("채집" / "보스 · 빠른 추격자" 등) |
| `secondsLeft` | `number` | 현재 단계 남은 초 |
| `upcomingThreat` | `{title, sub} \| null` | 다가오는 보스/대멸종 예고(하이라이트 배너) |
| `timeline` | `{progress, markers[]}` | 런 전체 진행 막대 + 보스/멸종 시점 마커 |
| `stageNumber`/`totalStages` | `number` | 단계 N / M |
| `eraLabel` | `string` | "시대 N / 5" |
| `isFinalEra` | `boolean` | 마지막 시대인가(정복 직전) |
| `level`/`xp`/`xpToNext`/`xpProgress` | `number` | 인런 레벨(=세대)·경험치·게이지 0~1 |
| `pickedCardNames` | `string[]` | 이번 런에 고른 카드 이름들(빌드 패널) |
| `interpAlpha` | `number` | 렌더 보간 비율(sim 30/s ↔ 화면 60fps 사이 메움) |
| `speed`/`paused` | `number`/`boolean` | 관전 배속(1/2/3)·일시정지(UI가 직접 set) |
| `metaLevelNow`/`rerollUnlockedNow` | `number`/`boolean` | 디버그 표시용 현재 메타 레벨·리롤 해금 |
| `environmentSummary()` | `string` | 이 맵 환경 한 줄 요약 |
| `width`/`height`/`areaScale` | `number` | 논리 해상도·면적 배율 |

### 4.4 디버그 전용(`?dev` URL에서만)

`debugSummon(kind)` 위협 즉시 소환 · `debugSetMetaLevel(n)` 메타 레벨 세팅 · `debugGrantMetaXp(n)` 진척도
재생용 · `debugReset()` 저장 진행도(레벨·챔피언) 초기화.

## 5. 현재 UI 컴포넌트(HTML 오버레이) 목록

각 컴포넌트는 `create*()` 팩토리로 만들고, `Game` 콜백/메서드에 배선된다(`src/main.ts`가 배선).
**UI를 새로 짠다면 이 목록의 역할을 그대로 대체하면 된다.**

> **이 표 읽는 법(핸드오프 완결성).** 각 행은 "현재 어떤 형식으로 무엇을 표시하는가(역할)" + "어떤 데이터를
> 받아 어떻게 처리·표시하는가(계약)"를 담는다. 맥락을 모르는 사람이 이 표만 보고 같은 컴포넌트를
> 다시 만들 수 있어야 한다. 형식·데이터가 복잡한 컴포넌트는 아래 §5.1처럼 별도 계약 절을 둔다.

| 컴포넌트(`src/ui/…`) | 역할(현재 형식) | 받는 데이터 / 처리·호출 |
|---|---|---|
| `hudPanel` | **상단 HUD(DOM 오버레이)** — 정보 카드·진행 타임라인·접이식 "종 안내" 범례 | 매 프레임 `update(HudData)` 로 표시, 새 런에 `reset()`. **상세 계약은 §5.1** |
| `lobby` | 타이틀/로비 | "시작" → `beginRun()`, "도감" → glossary |
| `presetPanel` | 시작 프리셋(종) 선택 창(외형 미리보기 + 페이지 넘김) | `onDraft`로 열림(`isChoosingPreset`), 탭 → `pickCard(i)` |
| `draftPanel` | 레벨업/시대보상 카드 3장 | `onDraft`로 열림, `pickCard`/`skipDraft`/`reroll`, `canReroll`로 리롤 버튼 표시 |
| `resultPanel` | 런 종료 요약 + 다음 행동 | `onResult`로 열림. "다음 시대로"→`continueToNextEra()`, "새 혈통"→`beginRun()` |
| `levelUpScreen` | **런 종료 진척도 화면**(경험치바 차오름 + 레벨업 + "새로 열림" 해금 하이라이트) | `onResult`의 `progress`를 애니메이션 |
| `momentOverlay` | 승리/정복/멸종 **순간 연출**(큰 글자·빛·암전) | `onResult` 직후 재생, 끝나면 진척도/결과로 |
| `creatureCard` | **개체 정보 카드**(좌하단) — 탭한 한 마리의 이름·기운·나이·**형질** | 월드 탭/‹›로 개체 선택. **개체별 게놈**을 보여줘 개체차 비교(§9) |
| `controls` | 우상단 컨트롤 바(배속·일시정지·멈춤 메뉴) | `speed`/`paused` set, `toLobby()` 등 |
| `buildPanel` | "설계도"(우측, 접이식) — 내 종 게놈·고른 카드 | `genome`/`pickedCardNames` 읽어 표시 |
| `glossary` | 용어·형질·생물 도감 | 로비·멈춤에서 열기 |

**공용 스타일·토큰:** `panelStyles.ts` 의 `:root` 가 색·활자·반경 토큰(§10·§4 참조)을 선언하고,
모든 컴포넌트가 `var(--*)` 로 공유한다. 좌하단 조작 열(한 마리 관찰=lime 입체 키, 줌 +/−=알약)은
`main.ts` 가 직접 만든다.

렌더 쪽(`src/render/`, 여전히 Pixi 캔버스): `worldView`(생물·지형·먹이·보스 스프라이트, 카메라,
선택 고리), `minimap`(지형 축소 맵 + 무리·보스·뷰포트 점 — 픽셀 맵이라 DOM 아님), `threatBanner`
(위협 예고 전광판), `effects`·`highlights`(사건 연출). 이 캔버스 레이어 위에 위 DOM 오버레이가 얹힌다.
(상단 HUD 는 예전엔 Pixi `render/hud.ts` 였으나 2026-07-08 DOM `ui/hudPanel.ts` 로 이관됨.)

### 5.1 상단 HUD — 현재 형식·표시 방식·데이터 계약 (완결 계약의 본보기)

> 이 절은 "맥락 없는 사람이 이것만 보고 HUD 를 다시 만들 수 있게"의 본보기다. 형식이 복잡한 다른
> 컴포넌트도 새로 손볼 때 이 틀(현재 형식 → 받는 데이터 → 처리 규칙)로 문서를 남긴다.

**현재 형식(무엇을 어떻게 표시하나).** 캔버스 위 DOM 오버레이 세 조각. 전부 3a 유리 패널·둥근 모서리:
- **정보 카드(좌상단 알약).** 1행 `내 종 N`(Jua) + `야생 M`(faint) · 우측 낮밤 점+라벨. 2행 상태줄
  (시대·단계·남은 시간·환경, 여러 줄). 데스크톱은 개체 수 추이 스파크라인(lime SVG). 아래 `Lv.N`(mono
  amber) + 경험치 진행바(amber). 사망이 생기면 맨 아래 "사망 …" 한 줄(red).
- **진행 타임라인.** 얇은 막대(모바일=카드 아래 full-width, 데스크톱=상단 중앙). 진행 채움 lime,
  보스 마커=red 탭+"보스" 칩, 종료(대멸종) 마커=ink 탭+"멸종" 칩.
- **범례(접이식 "종 안내").** 관전 중에만. 종별 색 점(내 종=lime 고리, 우호=청록, 야생 동맹=금빛)·이름·
  실시간 개체 수, 그 아래 먹이 색 점.

**받는 데이터(입력 계약).** `main.ts` 가 매 프레임 `hud.update(data)` 를, 새 런/다음 시대에 `hud.reset()` 을
부른다. `HudData` 는 모두 `Game`/`World` 에서 읽는다(HUD 는 sim 을 바꾸지 않는다):

```ts
interface HudData {
  world: World;        // playerPopulation·population, species[], entities[], deaths, daylight, dayPhase 를 읽음
  statusText: string;  // game 의 statusLine() 결과. 여러 줄(\n). "" 이면 로비 → HUD 전체 숨김
  level: number;       // 인런 레벨(=세대)
  xpProgress: number;  // 0~1 (경험치 진행바)
  timeline: { progress: number; markers: readonly { kind: "boss" | "extinction"; at: number }[] };
}
hud.update(data): void   // 매 프레임
hud.reset(): void        // 추이 표본·사망 집계·범례/타임라인 시그니처 리셋
```

**처리 규칙(받으면 어떻게 그리나).** 성능을 위해 값과 DOM 재생성을 나눈다:
- **매 프레임 텍스트·폭만** 갱신(개체 수·상태줄·낮밤·레벨·경험치 폭·타임라인 채움 폭).
- **시그니처가 바뀔 때만 재생성**: 타임라인 마커(markers 배열 변화 시), 범례 행(species 구성·접힘 상태 변화 시).
- 사망 알림은 48프레임마다 `world.deaths` 델타를 집계해 상위 2개만. 데스크톱 추이선은 8프레임마다 표본.
- `statusText === ""`(로비) → 전체 숨김. `statusText` 에 "카드 선택" 포함(드래프트) → 범례만 숨김.
- 탭 처리: 카드·범례는 `pointer-events:auto`(탭이 뒤 캔버스로 안 샘), 타임라인은 `none`(카메라 드래그 통과).

## 6. 렌더 루프와 배선(요지)

`src/main.ts`가 부팅 시:
1. Pixi `Application` + `WorldView`(+ `minimap`·`threatBanner`·`effects`·`highlights`) 생성, 캔버스를
   화면에 채운다(scale-to-fit). 상단 HUD 는 캔버스가 아니라 DOM 오버레이(`createHudPanel()`)로 만든다.
2. `Game` 생성, 위 UI 컴포넌트들을 만들어 콜백/메서드로 배선.
3. 매 프레임 `app.ticker`:
   - `game.update(deltaMS)` (관전 중이면 sim 진행)
   - 카메라 목표 계산(자동 추적/수동 팬·줌/개체 포커스)
   - `worldView.sync(game.world, game.interpAlpha, dtMS)` + `hud.update({...})`(§5.1) + 하이라이트 감지
   - 선택 개체가 있으면 `creatureCard.update({... en.genome.traits ...})`

즉 **UI 오버레이는 이벤트/상태 기반(콜백), 렌더는 매 프레임 pull 기반**이다.

## 7. 결정론·게놈 — UI가 알아야 할 sim 규칙

- **결정론:** 같은 시드 + 같은 선택 → 항상 같은 판. sim 안의 모든 무작위는 시드 RNG. UI/렌더는 결정론에
  영향 주면 안 된다(순수 연출만). `?seed=…`로 판을 재현할 수 있다(콘솔에 시드 로그).
- **게놈(Genome):** 버전 붙은 직렬화 구조. 형질 12종(속도·공격·시야·무리·대사·번식·식성 = 연속 생태 형질,
  수영·초음파·날개·독침·원거리 = 능력형 정체성). 값은 0~100(연속 형질은 카드로 200까지). UI는 이 값을
  읽어 표시만 한다(빌드 패널·개체 카드·도감).
- **카드(Card):** 게놈에 누적되는 형질 변화. 런 내 영구, 런 종료 시 리셋. 표시 수치 = 실제 적용값(연속
  형질은 카드 원값의 0.6배가 붙는다 — `effectiveDelta`로 이미 맞춰 표시).

## 8. 메타 진행(런 밖 영속 상태)

localStorage에 저장(`selpress_meta_v1`, `selpress_champions_v1`). sim 결정론과 무관(게임 밖 상태).

- **플레이어(메타) 레벨:** 런을 마칠 때마다 성적(도달 레벨·시대·정복)만큼 **누적 메타 경험치**가 쌓여
  레벨이 오른다. 런 종료 진척도 화면(`levelUpScreen`)이 경험치바를 채우며 레벨업을 연출.
- **언락 티어(메타 레벨 기준):** Lv2=다시 뽑기(리롤), Lv3=바다 개척자·헤엄, Lv5=하늘·날개, Lv7=원거리,
  Lv9=독 살갗, Lv12=초음파. 레벨이 오르면 프리셋·카드·리롤이 열린다(파워가 아니라 **선택지 확장**).
- **챔피언(비동기 생물):** 지난 런에서 활약한 내 종을 저장해 다음 런 세계에 "예전의 나"로 다시 등장.

## 9. 개체별 진화 — 현재 상태와 **관찰성 갭**(중요)

**2026-07-06 도입.** 예전엔 새끼가 종 기준선 게놈을 복제해 한 세대 안에서 다 똑같았다. 이제:
- 새끼는 **부모 게놈을 상속 + 연속 형질을 조금 변이**(독립 RNG) → 같은 무리 안에서도 개체가 갈리고,
  환경에 맞는 개체가 더 살아남는다(자연선택).
- 카드는 **살아있는 무리 전체 + 기준선**에 적용(플레이어가 방향을 쥔다).

**관찰 방법(현재):** "◎ 한 마리 관찰" 버튼 → 카메라가 한 마리를 따라가며 `creatureCard`에 그 **개체의
형질**을 표시. ‹ › 로 같은 무리의 다른 개체로 넘기면 형질 수치가 실제로 다르다(개체차 확인).

**갭(UI 개선 후보 — 사용자 피드백):**
1. 생물이 빠르게 돌아다녀 **누가 누군지, 누가 방금 태어난 앤지 구분이 안 된다.** → 신생아 표식(잠깐 빛나는
   고리), 개체 즐겨찾기/이름 고정 같은 것이 필요.
2. **변이가 미묘하다.** 현재 변이 강도가 작다(소수 개체 + 통과기준이 낮아 강도를 키우면 밸런스 테스트가
   노이즈로 흔들려서). 더 눈에 띄게 하려면 (a) 절대-통과기준 테스트를 "방향(고대사>저대사)" 기반으로
   견고화한 뒤 강도를 올리거나 (b) 개체차를 스프라이트에 더 크게 반영하거나 (c) 무리 형질 분포(평균·범위)를
   화면 어딘가에 수치로 보여주는 방법이 있다.
> UI를 다시 만든다면 "개체가 각자 다르고 진화한다"를 **한눈에 읽히게** 하는 게 이 게임의 정체성과 직결된다.

## 10. 한국어 UI 문구·톤 규칙(사용자에게 보이는 모든 말)

- **톤:** 차분한 자연 관찰 다큐/야외 도감(현재형 관찰, 담백·정확). 과장·감탄사(!)·귀여운 어미 지양.
  플레이어는 **관찰자·사육사**("지켜본다/기른다"), 조종자가 아니다.
- **쉬운 말만.** 생태/진화 전문 용어는 풀어 쓴다(적합도 필터→"잘 적응한 종만 통과하는 관문", 포식→"잡아먹기").
  게임 기본어 "형질·게놈·드래프트"는 쓰되 처음 한 번 풀이. 한자 약어 금지("死因" ❌ → "사망 원인" ⭕).
- **한글 사이 em dash(—) 금지.** 마침표·쉼표·줄바꿈으로. "이겨요" 대신 "승리".
- **보스 위협 예고는 반드시 읽혀야 한다** — "왜 졌는지 모르는데 졌다"가 되면 안 된다.
