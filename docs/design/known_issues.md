# 알려진 함정 / 반복 금지 (Known Issues)

> 버그·설계 함정·작업 실수를 발견하면 여기에 "증상 → 원인 → 재발 방지책"으로 남긴다.
> 게임 버그뿐 아니라 프로세스 실수(도구 오용, 커밋 누락 등)도 포함.
> 세션 시작 루틴에서 이 파일을 먼저 읽어 같은 실수를 예방한다.

### sim 효과를 넣으면 관련 시각(부채꼴·도감·범례)도 반드시 함께 갱신 (일부만 고치면 어긋남)
- 증상: 수풀 시야 감소를 behavior(sim vision)에만 넣고, 화면 시야 부채꼴(worldView playerVision)과
  대백과 시야 그림에는 반영을 빼먹음. → 수풀 안에서 실제론 시야가 줄었는데 부채꼴은 그대로라 "수풀서
  잘 쫓아다닌다"로 보이고, 도감 그림도 부정확. 사용자 질책("수정할 때 빼먹지 말라").
- 원인: 한 효과(지형×형질)가 여러 표면(sim 계산·부채꼴 렌더·도감 SVG/설명·범례)에 걸쳐 있는데 sim만
  고침. 시야 부채꼴은 종 게놈 "고정값"이라 개체 위치(수풀)와 무관하게 그려지고 있었다.
- 재발 방지책: **새 sim 효과를 넣거나 형질/지형 계산을 바꿀 땐 그 효과가 보이는 모든 표면을 체크리스트로
  훑는다** — ① sim 계산 ② 그 값을 그리는 렌더(부채꼴·오라·틴트 등, "고정값"으로 계산하던 곳은 개체별로)
  ③ 대백과(SVG 그림 + 수치 + 설명) ④ 범례/HUD. 렌더가 sim 과 같은 함수를 쓰게 하면(behavior 의
  grassVisionFactor 를 worldView 도 import) 어긋남이 원천 차단된다(시각=로직 1:1의 실천).

### GitHub Pages 첫 배포 시 configure-pages 권한 에러

- 원인: 워크플로 기본 `GITHUB_TOKEN` 권한으로는 Pages 사이트를 처음 만들지 못함.
- 재발 방지책: 레포 생성 직후 사용자 토큰으로 Pages 를 먼저 활성화한다 →

### sim 행동을 바꾸면 개체 수 스케일이 변해 절대 기준 밸런스 테스트가 깨진다
- 증상: 이동 로직(관성/목표 고정)을 매끄럽게 했더니 채집 효율이 올라 개체 수가 전반 상승,
  한파 대멸종 필터의 절대 통과기준(저대사<10) 테스트가 실패(저대사가 14로 생존).
- 원인: 게임 밸런스가 "절대 개체 수 기준"으로 잡혀 있어, 행동 효율이 바뀌면 분포가 통째로 이동.
- 재발 방지책: sim 행동을 바꾸면 무작정 상수만 만지지 말고 **임시 프로브 테스트로 분포부터 측정**
  (forage/poison/cold/heat/chaser/sanity 한 번에 console.log) 후 타깃 튠. 평상시에 영향 없는
  지점(예: 대멸종 전용 배율)을 골라 고치면 sanity/Phase3 회귀를 피한다.

### 종 다양성: 단일 먹이 자원에서 경쟁 배제로 금방 1~2종만 남음
- 증상: 야생종을 늘려도(3→6종) 수천 틱이면 1~2종(보통 내 종)만 남음. 시드 편차 큼.
- 원인: 모든 초식종이 같은 먹이 한 종류를 두고 경쟁 → 경쟁 배제 원리상 한둘만 살아남음.
  36마리로 시작하는 내 종이 동등 채집력으로 수적 지배까지 함.
- 완화책(현재): 생존 전략 분리(다산/저대사 장수/잡식/포식) + 야생 영역(클러스터) 스폰으로
  초·중반 공존을 늘림. 후반 붕괴는 못 막음.
- 근본 해결(적용 478e1d1): (1) 먹이 종류 분할 — food.kind 3종, 야생 초식종마다 다른 종류 전문이라
  서로 경쟁 안 함(전문종 공존). (2) 야생 주기적 이주 — 적은 야생종을 10초마다 소수 보충(멸종 바닥).
  결과: 4시드 전부 t750~t4000 내내 7종 전부 유지. 교훈: 단일 자원 경쟁 배제는 자원을 쪼개야 풀린다.

### 월드 스케일(areaScale) 도입 시 절대 상수를 빠뜨리면 그 축에서 분포가 붕괴
- 증상: 맵 3배(areaScale 9) 후 내 종은 늘고 야생만 단조 급감(t2000 야생 4%). 작은 월드(scale 1)는
  야생이 회복 진동(46%)하는데 큰 월드만 회복 못 함.
- 원인: areaScale 로 개체·먹이·cap·통과기준은 면적 비례시켰지만 **두 절대 상수를 빠뜨림** —
  ① 야생 보금자리 spread(72px): 큰 맵에서 좁은 점에 과밀 → 국소 먹이 소진 후 집단 아사.
  ② 이주 floor 4·batch 3: 큰 맵 종당 ~90 대비 4%라 보충이 무의미 → 줄어도 회복 못 함.
- 해결(4f0a45a): spread = 72·sqrt(areaScale)(길이라 제곱근), 이주 floor·batch = ×areaScale.
  큰 월드 야생 t2000 4%→48%(내 종과 균형).
- 재발 방지책: **월드 크기/스케일을 바꿀 땐 "모든" 절대 상수를 면적 기준으로 점검하라** — 개체수·
  먹이수·cap·통과기준(면적 비례) / 보금자리·이동거리 같은 "길이"(제곱근 비례) / 이주 바닥·보충량.
  하나만 절대값으로 남으면 그 축에서만 밀도가 어긋나 분포가 한쪽으로 쏠린다. 작은 vs 큰 월드 같은
  지표를 A/B 로 재서 "큰 월드만의 이상"을 가르는 게 진단의 핵심(원래 그런 것 vs 스케일 버그).

### 렌더 보간 없으면 sim 을 부드럽게 해도 화면은 드득거린다
- 증상: 행동 로직을 매끄럽게 고쳤는데도 생물이 잔떨림(특히 정지 화면/로비에서).
- 원인: sim 은 30스텝/초, 화면은 60fps. 보간 없이 sim 위치를 그대로 찍으면 두 프레임마다 점프.
- 재발 방지책: 고정 타임스텝 렌더는 prev/현재 위치 + alpha(스텝 진행률)로 보간한다
  (prevX/prevY 를 스텝 시작에 기록, game 이 alpha 노출). 약 1스텝 지연은 관전엔 무해.

### ✅ 생물 떨림(jitter) — 해결됨 (2026-06-29, 사용자 폰 확인)
- **증상**: 스프라이트가 좌우로 빠르게 진동 회전(제자리·이동 중 둘 다). 폰 실기에서만(개발 머신 비재현).
- **최종 원인(확정)**: 회전 목표를 매 스텝의 미세 이동 방향 `atan2(e.x-e.prevX, e.y-e.prevY)` 으로 잡음.
  저속/조향 흔들림이면 한 스텝 방향이 좌우로 뒤집히고, 데드존(18°)은 작은 변화만 막지 방향 "뒤집힘"
  (큰 각)은 통과 → 부들부들 회전. **위치가 아니라 회전이 주범**(제자리에서도 미세 변위 방향이 노이즈).
- **해결(커밋 62e9ac1, `render/worldView.ts`)**: 회전 목표를 **헤딩 벡터 저역통과**로. 개체별 헤딩 벡터를
  `hd += (스텝변위 - hd) * headK` 로 평활해 `sp.rotation` 이 `atan2(hd)` 를 데드존+이징으로 따라간다.
  방향이 좌우로 튀어도 평균이 0 근처로 수렴 → 각도 안정. 진짜 전환만 서서히 따라감. `?head=<0~1>` 튜닝.
- **진단 교훈(가장 중요)**: 떨림은 `?norot`(회전 고정)로 **회전/위치부터 가르는 게 첫 단추**. 사용자가
  "좌우로 진동하는 회전"이라 정확히 묘사해준 게 결정타였다. 한때 ?norot 에도 떤다고 위치로 오진해 헛돈 게
  2~3세션. 증상 묘사를 끝까지 정확히 받는 것이 진단의 핵심.
- **막다른 길(반복 금지)**: ① sim 파라미터 튠(관성 조향/목표 고정/arrive 감속/cohesion comfort/wander
  보존) ② 위치 보간(prevX/prevY+alpha) ③ 위치 저역통과(dispPos, `?smooth=0.01` 거의 얼림에도 떨림 →
  위치 진폭 무관 증명) ④ roundPixels(위치 sub-pixel용) ⑤ 회전 이징 프레임률 독립화·데드존만.

  벗어난 정도로 ramp(26) 동안 0→최대(낙오자만 모음). huddle(개체수 기반)은 불변→추위 밸런스 유지.
  교훈: 격자 기반 이웃 질의의 COM 은 셀 경계에서 양자화 점프한다 — 그걸 직접 추종하면 떨린다.
- **마지막 시도(c9b1961): sim 레버 소진 → 렌더 평활.** 사용자 "노랑·주황 아직 와리가리". 임시 프로브로
  종별 reversal%(스텝당 속도 방향 반전율)를 객관 측정하니 폰 체감과 일치(전문종 2~3% vs 내 종 0.5%).
  cohesion/wander/steerTurn/먹이밀도/arriveRadius 를 다 쓸어봤으나 **어느 것도 전문종을 안정적으로 못
  내림**(먹이 촘촘화는 더 자주 먹어 오히려↑, 측정도 적은 개체수라 노이즈 큼). 결론: **전문종 떨림은
  적은 개체가 드문 전문 먹이를 먹는 생태의 본질** — sim 으로 못 없앤다. → 렌더 전용 위치 저역통과
  (`worldView.dispPos`, 프레임률 독립, ~50ms 지연)로 화면에서만 흡수. sim 불변(밸런스/결정론 무관).
  `?smooth=<0~1>` 로 강도 튜닝(작을수록 부드럽고 잔상↑, 1=끔).
- **결정적 진단 2 (2026-06-29): 위치 평활도 막다른 길.** 사용자가 `?smooth=0.01`(거의 얼림)에도
  "떨려" → 떨림은 **위치 진폭과 무관**. 생물은 이미 캐시 스프라이트(풀 재사용)라 위치를 아무리
  굳혀도 떨린다면 원인은 위치가 아니다. → 남는 원인 = **서브픽셀 안티앨리어싱 깜빡임**: 텍스처가
  resolution:3 슈퍼샘플+antialias 인데 스프라이트가 비정수 위치에서 매 프레임 미세 이동하면 다운샘플
  격자가 어긋나 가장자리가 부들거린다(평활은 sub-pixel 이동을 0으로 못 만든다).
- **수정 시도(roundPixels, 빗나감): `sp.roundPixels = true`** — 위치 sub-pixel 겨냥. 무해하나 회전
  진동엔 효과 없음(원인이 위치가 아니었음). 둬도 됨.
- **결정적 정정(2026-06-29, 사용자): "떨림 = 좌우로 빠르게 진동하는 회전". 위치/이동이 아니라 회전이
  문제.** 제자리·이동 중 둘 다. → 두세 세션 위치만 의심한 게 헛발질이었던 진짜 이유.
- **원인(확정): 회전 목표가 "매 스텝의 미세 이동 방향"(`atan2(e.x-e.prevX,…)`)이라 노이즈.** 저속/조향
  흔들림이면 한 스텝 방향이 좌우로 뒤집히고, 데드존(18°)은 작은 변화만 막지 방향 "뒤집힘"(큰 각)은
  통과시켜 스프라이트가 부들부들 회전. 제자리에서도 미세 변위 방향이 노이즈라 그렇다.
- **수정(2026-06-29, 폰 검증 대기): 회전 목표를 "헤딩 벡터 저역통과"로.** 개체별 헤딩 벡터를
  매 프레임 `hd += (스텝변위 - hd)*headK`로 평활 → 방향이 좌우로 튀어도 평균이 0 근처로 수렴해
  각도가 안 바뀐다. 진짜 방향 전환만 서서히 따라감. `?head=<0~1>`(작을수록 안정·전환 느림, 1=끔).
  **이래도 떨리면** headK 더 낮추거나(예 ?head=0.04), sim 의 속도벡터 자체 진동을 behavior 에서 봐야.
- 진단·튜닝 토글(src/debug.ts, 좌하단 HTML 배지에 값 표시): `?norot`(회전고정·회전/위치 切分),
  `?nointerp`(보간끔), `?showalpha`, `?dz`/`?rotk`(회전), `?smooth`(위치 평활).

### 반응형 벽 회피(avoidWalls)는 진동을 만든다 — headless 프로브가 진동을 못 잡아 폰과 어긋남
- 증상: 이동 차단(지형) 후 생물이 벽에 막히면 못 돌아감. `avoidWalls`(가려는 방향이 막히면 desired 를
  통행 방향으로 회전)를 넣었더니 폰에서 끼임이 **오히려 더 늘었다**(사용자). 그런데 headless
  프로브("변위<0.1=정지")로는 끼임이 줄어든 것으로 측정돼 정반대 결론이 나왔다.
- 원인: ① avoidWalls 가 매 틱 좌우(±0.35rad)를 독립 탐색 → 벽 옆에서 선택 방향이 휙휙 뒤집혀
  desired 가 좌우로 진동 → 제자리 맴돌기. ② 반응형 조향은 오목 지형·막다른 만에서 local minima 에
  빠져 못 빠져나옴(근본 한계). ③ 프로브가 "정지"만 재서 **진동(변위는 있음)을 "안 끼임"으로 분류** —
  떨림 사건과 똑같은 맹점.
- 해결(7d653d7): 타일 격자 BFS 길찾기로 교체(`terrain.lineOfSight` 직진 우선 + 막히면 `findPath`
  경로 추종 + funnel 단축, 경로는 entity 캐시). desired 방향을 흔드는 게 아니라 "어디로 갈지"를
  격자에서 풀어 진동·local minima 가 없다. cohesion 도 무게중심이 벽 너머면 끔.
- 재발 방지책: **시각 현상(떨림/진동/맴돌기)을 "정지" 같은 단일 지표로 판단하지 말 것.** 프로브에
  방향 반전율(벽 옆 변위가 한 틱 전과 거꾸로) 같은 진동 지표를 함께 재고, 최종 판단은 폰 눈.
  정적 장애물(지형) 회피에 반응형 조향은 부적합 — 격자 길찾기가 정답.

### 우호 친척 종을 내 종 "근처"에 스폰하면 소수 밸런스가 깨진다 (근처 동거 불가)
- 증상: 친척 종(우호)을 내 종 보금자리 근처에 두려 하니(옆 동네 offset·먹이 분리·수↓ 다 시도),
  Phase 5 통과기준 테스트가 반복 실패 — 특히 외톨이 사냥꾼·그림자 매복자 보스에서 내 종이 0~2로
  전멸(통과기준 3 미만). 세션 2·3 두 번 프로브로 재현.
- 원인: ① **무리 cohesion 이 종 무관 이웃(grid.neighborhood COM)** — 친척이 내 종 무리에 섞이면
  COM 에 친척이 포함돼 내 종 뭉침이 흐트러진다. 외톨이/매복 보스는 "뭉치면 생존"이라 결속이 깨지면
  치명적. ② 소수 개체 + 통과기준 3(빡빡한 마진)이라 국소 먹이를 조금만 함께 먹어도 hi 가 3 미만.
  offset·먹이 분리(foodKinds)·수↓로도 "섞이면" ①이 남아 근본 해결 안 됨.
- 해결(채택): **친척은 맵의 독립 영역에 스폰**(야생종처럼). 떨어져 살되 이동하다 만나면 friendly 라
  서로 사냥·도망 안 해 자연스레 섞인다(스포어식 우호). 근처 동거 콘셉트는 버림.
- 재발 방지책: 소수 개체 게임에서 **새 종을 내 종과 같은 자리에 두지 말 것.** cohesion 이 종 무관이라
  무리가 섞여 결속·먹이가 교란된다. "같이 사는 느낌"은 독립 스폰 + 이동 중 조우(friendly)로 충분.
  근처 동거를 살리려면 cohesion 을 같은 종(+우호)만 보게 바꿔야 하는데, 그건 무리 역학 전면 재튠이라
  비용이 크다(현재 안 함).

### 형질 스케일을 정수화(0~100 반올림)하면 야생종 형질이 미세 달라져 밸런스가 이동한다
- 증상: 형질을 0~1 소수 → 0~100 자연수로 전환(값 ×100, 계산은 ÷100 정규화라 수학적 동일). 내 종
  게놈(default 50·카드 정수)은 정확한데, 보스 통과기준 테스트 일부(약탈자·매복자)가 깨졌다.
- 원인: randomGenome·야생 드리프트가 형질을 반올림(clampTrait)하면서 야생종 형질이 스케일 전과
  미세하게 달라졌다 → 야생 생태(먹이 경쟁)가 변동 → 내 종 개체수가 통과기준 경계에서 이동. 아슬한
  케이스(약탈자 lo·매복자)가 그 노이즈로 넘어갔다. 계산은 동일해도 "입력값의 이산화"가 sim 을 흔든다.
- 재발 방지책: 스케일/정수화처럼 형질 표현을 바꿀 땐 밸런스가 "수학적으로 동일"이라도 이산화로
  분포가 이동할 수 있다 → 통과기준 테스트를 프로브로 재확인하고, 개체 시뮬 + 소수 개체라 단일 시드
  형질 게이트가 원래 아슬한 시련은 "카운터 방향(hi>lo)·kill 발생"으로 견고화(엄격한 lo<기준은 노이즈).

### HUD 고정 위치 요소는 "가변 길이 텍스트"와 같은 줄에 두지 말 것 (우측 고정 위젯↔긴 라벨 겹침)
- 증상: 정보 박스 둘째 줄(상태) 우측에 낮밤 타이머(노란 점+라벨)를 고정했는데, 상태 첫 줄이
  긴 시련 이름("그림자 매복자"·"외톨이 사냥꾼")일 때 그 위젯과 겹쳐 보였다(폰).
- 원인: 좌측 텍스트는 내용에 따라 가로로 늘고, 우측 위젯은 절대 좌표 고정. 같은 y(줄)에 두면
  텍스트가 길어지는 순간 침범한다. 세션 8엔 개체 수(첫 줄, 짧음) 기준으로 타이머를 둘째 줄에
  내렸는데, 이후 그 둘째 줄에 긴 라벨(단계·시련 이름)이 들어오며 반대로 겹쳤다.
- 해결(aff577a): 타이머를 "짧고 안정적인" 줄(개체 수 = 소수 개체라 짧음) 우측으로 옮겨 세로
  분리 + 상태 첫 줄에서 "단계 N/M ·" 프리픽스 제거(타임라인 막대가 대신 보여줘 중복).
- 재발 방지책: 우측 고정 위젯은 **그 줄의 좌측 텍스트가 최대로 길어져도 안 닿는지**로 자리를
  정한다. 길이가 가변인 텍스트(단계·종·시련 이름 등) 줄에는 고정 위젯을 얹지 말고, 길이가
  거의 일정한 줄(개체 수)이나 별도 영역에 둔다. 좌표로 "좌측 끝 x < 위젯 시작 x"를 계산해 마진 확인.

### 시각(이펙트)은 실제 로직과 반드시 1:1 로 일치시킨다 (전역 로직을 국소 이펙트로 그리지 말 것)
- 증상: 보스 6종 중 5종(독안개·사나운 무리·약탈자·외톨이 사냥꾼·그림자 매복자)이 **전역 솎기/흡수**
  (위치 무관, 매 틱 모든 개체를 확률로 솎음)인데, 시각은 "떼(작은 점 뭉치)"·보스 점을 특정 위치에
  그려 놨다. → "저기 떼가 있는데 왜 여기 있는 내 애가 죽지?" 로직과 이펙트가 따로 놀아 사용자가 혼란·불신.
- 원인: 로직(전역)과 시각(국소 개체)을 각각 만들고 맞추지 않음. 개체가 없는 현상(전역 재난)에 개체(점)를
  그렸다. 게다가 "보스"라는 한 이름에 개체형(추격자)과 전역 재난(독안개)을 섞어 불러 더 헷갈림.
- 재발 방지책(원칙): **새 메커니즘을 만들 때 시각을 로직과 1:1 로 설계한다.**
  · 전역 효과(위치 무관) → 화면 전체 표현(전체 틴트/맥동, 사방에서 벌어지는 kill 이펙트). 개체(점) 금지.
  · 국소 효과(특정 위치) → 그 위치에만 표현.
  · 개체가 실제로 존재하는 것만 개체로 그린다(추격자=점+즉사 반경, 도망 대상).
  · 이름도 로직에 맞춘다: 실제 개체=보스, 전역 재난=시련. "무엇이 어떻게 죽이는지"가 화면과 말에서
    일치해야 "왜 졌는지"를 안다(기획서 §7 가독성). 로직을 먼저 정하고 시각·용어를 거기에 맞춘다.

### 새 통행/감각 파라미터(canFly 등)를 배선할 때 "기존 기본값에 의존하던 호출"을 바꾸면 밸런스가 이동한다
- 증상: 날개(비행) 형질로 terrain 통행 함수에 `canFly` 파라미터를 추가하며, cohesion 판정의
  `lineOfSight` 호출도 `(…, canSwim)` → `(…, canSwim, canLand, canFly)`로 "더 정확하게" 바꿨다.
  그랬더니 내 종/야생이 날개 0(비행 안 함)인데도 독안개 통과기준 테스트가 경계에서 실패(lo=base=1).
  고산 먹이(독립 rng)를 0으로 꺼도 실패해 "고산 먹이가 원인이 아님"이 드러났다.
- 원인: 그 `lineOfSight` 호출은 원래 `canLand`를 **생략(기본값 true)**했다. `canLand`를 명시하니 물
  전용 종(swimming≥90, 물고기 떼)의 `canLand`가 false가 돼 cohesion 판정이 바뀌고(육지 너머 무리를 안
  봄), 다종 생태의 궤적이 이동 → 내 종 개체수가 경계에서 넘나들었다. 날개와 무관한 "물 전용 종 동작
  변화"가 진짜 원인.
- 재발 방지책: 새 통행/감각 파라미터를 배선할 때 **"기존에 기본값에 의존하던 호출"은 그 기본값을 유지한
  채 최소로만 바꾼다.** 정확성을 위해 인자를 더 넘기고 싶어도, 그 인자가 특정 종(물 전용 등)에서 기존과
  다른 값이면 밸런스가 이동한다. 밸런스와 무관한 부수 판정(cohesion 등)은 원래 호출을 보존하는 게 안전.
  진단 팁: **새 자원(고산 먹이 등)을 의심하기 전에 그 자원을 0으로 꺼 원인을 빠르게 가른다**(끄고도
  실패하면 그 자원이 원인이 아니다).
