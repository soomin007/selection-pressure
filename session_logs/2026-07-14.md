# 2026-07-14 세션 로그

## 요약
백로그 최상단 **무리 성향(herding) 재검토**를 처리했다. 사용자 판단("무리 성향은 쓰레기 특성 아닌가")이
정확했음을 프로브로 확인하고, herding 에 처음으로 진짜 값을 주는 **무리 방어**를 넣었다. 뭉쳐 있으면
포식자가 그 무리를 **아예 표적으로 삼지 않는다**. 다산 초식 도달 2.0 → 4.0, 다른 프리셋은 불변.

가는 길에 **두 번 헛짚었다.** 둘 다 "멀쩡한 코드를 짜고도 아무 일도 안 일어나는" 종류라 known_issues 에
원인과 함께 남겼다.

## 주요 변경
- `src/sim/behavior.ts` — `chooseGoal` 의 먹잇감 후보에서 `herdShielded` 개체 제외. 쫓던 먹잇감이 무리로
  돌아가도 놓아준다(히스테리시스가 무리 한가운데까지 쫓아 들어가지 않게). 순수 규칙 `herdShieldedBy` export.
- `src/sim/spatialGrid.ts` — `countMatching`(반경 안 같은 종 이웃 수) 추가. 기존 `neighborhood` 는 종을
  안 가려(3×3 칸 전체) 무리 방어에 못 쓴다.
- `src/sim/params.ts` — `herdShieldThreshold` 85 · `herdShieldRadius` 46 · `herdShieldNeighbors` 2.
- `src/game/cards.ts` — 다산 초식 프리셋 herding 78 → 92(방패 임계를 넘기려는 값 — **둘은 한 쌍**).
- `src/ui/glossary.ts` — 무리 성향 도감을 "보온" 위주에서 "무리 방어" 위주로 고쳐 씀.
- `src/sim/world.test.ts` — 규칙 4건(두 조건 다 필요 · 낙오자는 방패 없음 · 야생 전 종 배제 · 카운터 방향).

## 결정 사항
- **표적 제외(이진)이지 확률 감소가 아니다.** 처음엔 "이웃이 많을수록 사냥당할 확률이 떨어진다"를 물기
  판정(`biteOutcome`)의 즉사 확률·피해에 저항을 곱해 구현했다. **효과가 0 이었다** — 저항 50% 를 걸어도
  잡아먹힘이 29→22 에 그치고 도달·승률이 안 변했다. 이유: **잡히는 개체는 이미 무리에서 떨어진 낙오자**라
  이웃 보정이 애초에 안 걸린다(전체 평균 저항 51% 만 보고 "잘 걸린다"고 오판했다). 물린 뒤가 아니라
  **물리기 전**, 표적 선택 단계에서 통째로 빼야 무리가 산다.
- **임계 85 는 야생 배제선이다.** 야생 초식은 herding 60~72 로 시작하지만 포식 압력을 받으면 한 런에
  최대 84 까지 진화한다(프로브). 임계를 75 에 뒀더니 긴 런에서 **야생 초식이 방패를 얻어** 균형 잡식이
  사냥감을 잃고 무너졌다(도달 6.1 → 5.5). 85 는 그 진화 최댓값 위 — 야생은 못 닿고 플레이어 무리 빌드만 닿는다.
  덕분에 다른 프리셋이 **소수점까지 불변**이다(완전 격리).
- **cohesion 은 건드리지 않는다.** "먹이 쫓는 중엔 뭉침을 약화"해 봤더니 **역효과**였다(무리가 흩어져
  잡아먹힘 30 → 49). 뭉침 자체가 이미 포식 방어로 작동하고 있었다는 뜻. 되돌렸다.
- **무리 채집은 폐기.** "뭉치면 채집 수입 +60~120%"를 만들었으나 효과 0(도달 3.4 vs 3.6, 승리 0/8).
  사인 집계를 보니 **굶주림 사망이 4~7 뿐**이고 잡아먹힘이 24~34 였다 — 먹이는 병목이 아니었다.
  번식이 적었던 건 에너지 부족이 아니라 **번식 전에 잡아먹혀서**였다. 인과를 거꾸로 읽었다.

## 세션 2 — 게놈 v7: 무리 성향 강등 + 몸집·은신 신설 (사용자 방향)

### 요약
사용자 질문: *"이렇게 하는 게 최선인가? 아예 무리짓기는 초식용 특성이나 하이에나 같은 특정 육식 라인에만
넣고, 새로운 형질을 넣을 만한 건 없나?"* — 코드가 이 직관을 확증했고, 게놈 v7 로 재편했다.
**무리 성향을 능력 형질로 강등**(기본 0, 카드로만)하고 그 자리에 **몸집(size)·은신(camouflage)**을 넣었다.

### 왜 herding 을 강등했나 (데이터)
herding 은 값 형질 슬롯 하나(기본 50)를 차지하면서 **여덟 프리셋 중 하나에서만** 실제로 작동했다:
- 다산 초식(92) = 무리 방어 ✓ / 균형 잡식(58)·독 살갗(68) = 임계 85 미달 → 없음
- 육식·정찰·바다·하늘·원거리(50) = 없음
- **하이에나 라인(무리사냥)은 이미 있었으나 죽어 있었다** — 이중 임계(diet>70 AND herding>55)라
  육식 사냥꾼 프리셋조차 diet 68 로 문턱 아래였다.
나머지 빌드에선 뭉치느라 먹이 탐색만 좁아지는 **순손해**. 결정적 증거: 육식 사냥꾼이 herding 을 잃자
**오히려 강해졌다**(도달 5.6 → 6.1).

### 주요 변경
- `src/sim/genome.ts` — `TraitsV7`(size·camouflage), `GENOME_VERSION` 7, v6→v7 마이그레이션.
  `defaultGenome`: herding 50 → 0, size 50, camouflage 0. `MUTABLE_TRAITS`: herding 빼고 size 넣음.
- `src/sim/behavior.ts` — `sizeDev`/`sizeSpeedFactor`/`sizeDrainFactor`/`sizeFertilityFactor`/
  `effectiveCamo`/`camoVisionFactor`. `biteOutcome` 이 **공격력 차 + 몸집 차**를 본다.
  은신은 포식자의 표적 탐색에서 시야 반경만 줄인다.
- `src/sim/species.ts`·`world.ts` — 네 생성 루프와 진화 드리프트가 새 형질을 **rng 없이** 설정.
- `src/game/cards.ts` — 「무리 본능」을 **관문 카드**로 승격(전설), 「보호색」·「커다란 몸」·
  「작고 날쌘 몸」·「그림자 무늬」 추가. 무리 강화 카드는 `requiresTrait` 로 무리 종에게만.
- `src/render/worldView.ts` — **몸집 = 스프라이트 크기**, **은신 = 흐릿함**(최대 45%).
- `src/ui/traitDisplay.ts`·`glossary.ts` — herding·camouflage 를 능력형(없음/보통/강함)으로.

### 결정 사항
- **밸런스 보존의 열쇠는 "기본값이 중립"이다.** 몸집 50 에서 모든 보정이 **정확히 0**(±(size-50)/50 에
  곱한다), 은신 0 이면 영향 없음. 그래서 v7 을 통째로 얹고도 305 테스트가 그대로 통과했다.
- **관문 카드는 한 장으로 능력을 열어야 한다.** 「무리 본능」이 +18 이던 시절엔 여러 장을 모아야 겨우
  켜지는데 설명은 "뭉친다"였다 — 거짓말이다(날개가 +42 라 아무 일도 안 하던 함정과 같은 계열).
  이제 `herding = herdShieldThreshold + 3`.
- **몸집과 은신은 한 축의 양끝이다.** 큰 몸은 못 숨는다(`effectiveCamo` 가 몸집으로 감쇠). 커져서
  버티거나, 작게 숨거나 — 둘 다는 안 된다. 프로브로 확인됨(은신+큰몸은 은신이 죽고 몸집만 남는다).
- **은신은 초음파를 못 속인다.** 눈을 속이는 것이지 소리를 지우는 게 아니다(감각 축끼리의 가위바위보).

|---|---|---|
| 균형 잡식 | 5.3 | **5.3** |

| 육식 사냥꾼 | 5.6 | 6.1 |

### 튜닝에서 배운 것 — 대가를 겹치면 함정 카드가 된다
몸집에 대가 셋(느림 0.25 · 대식 0.4 · 저번식 0.4)을 걸었더니 **큰 몸이 순손해**였다(몸집 85 초식이
8시드 중 7 멸종). 잡아먹힘은 27→15 로 줄어 방어는 분명히 작동했는데, 대가 셋이 **곱해져** 이득을

## 세션 3 — v7 폰 피드백 반영 (몸집 가시성·정점·성장 곡선·초음파)

### 요약
폰에서 v7 을 만져 본 사용자 피드백을 연달아 처리했다. **몸집이 안 보이던 게 내 설계 실수**였고
(카드 +24 가 실제로는 +14 만 들어가 스프라이트가 11% 만 커졌다), 그 김에 성장 곡선·난이도·정점
보상까지 손봤다. 사용자가 던진 새 방향 다섯은 내일 작업으로 정리했다(backlog 최상단).

### 주요 변경 (커밋 순서)
1. `6c8dae6` — 몸집 가시성(GROWTH_TRAITS 에서 size 제외 + 스프라이트 폭 0.4→0.62 → **한 장에 30%**),
   레벨업 완화(xpBase 24→40 · per 14→22, 드래프트 7→5.4회), 대백과 목차 + 뒤로가기 스크롤 복원,
   로비에서 ?dev 패널 숨김.
2. `1bfdb0f` — 순수 육식 "채집 절벽" known_issues 기록.
3. `2cad09e` — **정점(100 보상)** + **상한 근접 감쇠** + **난이도 복리** + 거인 카드 2종에 실제 몸집
   (레거시 `CARD_BODY_SCALE` 제거) + **초음파가 시야를 통째로 가져감** + 야생종 몸집 다양성 +
   경고 배너 wordWrap(폰에서 잘리던 문제).
4. `c2e60ed` — 쌍둥이 rng 함정 · 포식자 몸집이 밸런스 기준점임을 known_issues 에 기록.

### 결정 사항
- **몸집은 성장 스케일(×0.6)을 씌우면 안 된다.** 50 을 중심으로 **양방향**인 축이라 "천천히 자라는"
  스케일을 씌우면 카드 한 장으로 아무것도 안 보인다. 값 형질이라고 다 같은 규칙을 쓰면 안 된다.
- **레거시 `CARD_BODY_SCALE` 제거.** "외형만 키우는 별도 배율"은 몸집 축이 생기기 전의 임시방편이었다.
  두 축(외형/시뮬)을 따로 두면 언젠가 어긋난다 — 이제 몸집 하나에서 둘 다 나온다.
- **감쇠와 정점은 한 쌍이다.** 100 을 비싸게 만들었으면(감쇠) 닿았을 때 보답이 있어야 한다(정점).
  정점은 **수치를 더 키우는 게 아니라 그 형질의 약점을 지운다**(속도=험지 면제, 시야=밤·수풀 면제,
  공격력=체급 무시 규칙 면제, 번식력=문턱 완화). 한 시대엔 거의 못 닿는 **장기 목표**가 됐다.
- **난이도는 복리여야 한다.** 카드 성장은 곱셈처럼 쌓이는데 위협만 덧셈으로 오르면 후반이 시시해진다.
- **초음파는 시야보다 순수하게 우월하다**(전방위 + 어둠·수풀 무시) → 둘 다 가지면 시야가 무의미해진다.
  그래서 관문 카드가 시야를 0 으로 만든다(박쥐는 눈이 퇴화했다). 이제 진짜 트레이드오프다.
- **야생 포식자의 몸집은 건드리지 않는다(50 고정).** 64 로 키웠더니 sizeBiteWeight 때문에 내 종을 무는
  즉사 확률이 0.64 → 0.93 으로 뛰어 대멸종 필터가 깨졌다. 포식 판정의 한쪽 항이라 밸런스의 기준점이다.

### 튜닝에서 배운 것
- **정점 보상을 "새끼 하나 더"(쌍둥이)로 만들면 안 된다** — rng 를 두 번 더 소비해 스트림이 밀리고
