# 2026-07-10 세션 로그

## 요약
새로 받은 「핸드오프 패키지」의 드래프트(형질 선택) 화면 스펙 v1.0 을 구현해 적용했다. 먼저 스펙 §4 의
"계속 움직이는 시뮬레이션 위에" 문구를 사실에 맞게 고쳤고(월드는 멈추고 캔버스만 살아 있다), 스펙이 전제하지만
코드엔 없던 **희귀도** 개념을 뽑기 확률과 함께 도입한 뒤 전체 화면 드래프트 UI 를 새로 짰다. 실제 브라우저로
몰아서 검증하다 실제 버그 두 개(HUD 가 안 숨음, 낮은 폰에서 히어로가 헤더·카드를 덮음)를 잡았다.

## 결정 사항
- **드래프트 중 월드는 멈추고 캔버스는 계속 그린다.** 스펙 문구가 (가) 시뮬 로직 진행과 (나) 화면 움직임을
  한 문장에 뭉쳐놨다. (가)는 반드시 멈춰야 한다 — 제한 시간이 없는 화면이라 고민하는 동안 생물이 죽으면 안 되고,
  스텝 수가 플레이어가 카드를 본 시간에 좌우되면 시드 시뮬의 결정론이 깨지며, `updateXp()` 가 계속 돌아
  드래프트가 열린 채 두 번째 레벨업이 겹친다. 이미 `game.ts` 의 `phase !== "watch"` 가드가 이걸 막고 있었다.
  (나)는 `blur(16px)` + 하단 0.95 그라데이션 아래에선 거의 안 보이는데 모바일 재블러 비용만 크다. 생동감은
  선명한 전경인 §5 히어로가 이미 담당한다. 로비는 반대로 앰비언트 스텝을 유지한다(결정도 결과도 없는 화면).
- **비트맵 캡처 대신 캔버스에 CSS 필터.** 마지막 프레임을 캡처하면 카메라·리사이즈 대응이 지저분해지고
  선명도를 잃는다. `scale(1.06)` 으로 블러 가장자리 번짐을 화면 밖으로 민다(스펙의 "사방 24px 확장" 대응).
- **희귀도는 장식이 아니라 규칙에 연결했다.** 배지에 "전설"이라 쓰고 균등하게 뽑으면 표시가 거짓말이 된다.
  등급을 카드 리터럴에 흩지 않고 `CARD_RARITY` 한 곳에 모아 분포를 보며 튜닝한다. 현재 전설이 3장 중 하나로
  뜰 확률 ≈ 5%(한 런에 한 번 볼까 말까). 가중치만 바꾸면 체감이 바뀐다.
- **히어로 배율은 transform, 등장 연출은 안쪽 엘리먼트.** 한 엘리먼트에 두면 키프레임의 transform 이 배율을
  통째로 덮어쓴다(스펙 §8 함정 2와 같은 종류). 스펙은 데스크톱에만 배율을 걸어 낮은 폰에서 깨진다 — 양쪽 다 건다.

## 검증
- `?seed=100` 이면 첫 드래프트 첫 장이 전설(치타의 다리)이다. 콘페티·금빛 플래시·3px 링 경로를 실제로 태울 때 쓴다.
  (신규 프로필에서 잠금 해제된 전설은 `cheetah` 하나뿐 — 나머지는 메타 레벨 3~12 해금.)
- playwright 로 420×880·1280×900·360×640·1280×600 을 몰아 확인: 캔버스 블러, HUD·컨트롤 숨김, 카드 3장,
  데스크톱 3등분 폭 동일(§8 flex-basis 함정), 전설 배지·콘페티 16개, rare-flash `backwards` 라 선택 링이 살아 있음,
  팝업 가로 중앙(§8 transform 함정), 낮은 창에서 CTA 안 밀림·히어로 안 겹침.
- `npm run typecheck` · `npm test`(151) · `node scripts/smoke.mjs` 모두 통과.

- 스펙 §11 의 조정 파라미터(블러·밝기·딤·바운스·뜸·콘페티 수)는 `ui/rarity.ts` 의 `DRAFT_TIMING` 과 CSS 상수로
  흩어져 있다. 폰에서 튜닝할 일이 잦아지면 URL 쿼리(`debug.ts`)로 빼는 걸 검토.
- 시대 보상 드래프트(`boostCard` 로 강화된 카드)도 같은 화면을 쓴다. 강화 카드는 수치만 커지고 등급은 원본을
  따르므로, "강화됨" 표식이 필요한지 폰에서 확인.
