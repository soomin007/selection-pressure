# 2026-07-11 세션 로그

## 요약
backlog 최우선 항목이던 **식성(diet) 효율 연동**을 구현했다. 그동안 `foodEnergy`·`predationEnergy`가
식성과 무관한 상수라 잡식(diet 50)이 사냥·채집을 둘 다 100% 효율로 해, 특화는 순손해였고 「포식/초식
본능」 카드와 특화 프리셋이 사실상 함정이었다. 섭취 효율을 diet 에 연동하되(제너럴리스트 페널티), 곡선을
특화 끝에서 평평하게 둬 야생 생태·대멸종 필터를 보존했다. 페널티 폭은 프로브로 필터 제약 하에 튠했다.

## 주요 변경
- `src/sim/behavior.ts` — `grazeEfficiency(diet)`·`huntEfficiency(diet)` 순수 함수 추가(export).
  섭취 지급(`foodEnergy × grazeEfficiency`)·사냥 지급(`predationEnergy × … × huntEfficiency`) 두 곳에 적용.
- `src/sim/params.ts` — `dietSpecializationPenalty`(0.3) 추가. `foodEnergy`·`predationEnergy` 주석 갱신.
- `src/sim/world.test.ts` — 효율 규칙 단위 테스트 5건(특화=1.0·잡식 페널티·문턱 정렬·특화 유인·단조성).
- `docs/design/backlog.md` — "식성 효율 연동" 착수 항목 삭제, "폰 체감 튠" 후속 항목으로 교체.

## 결정 사항
- **곡선을 특화 끝(순수 초식 diet≤35 / 순수 육식 diet≥70)에서 1.0 으로 평평하게, 잡식 구간(35~70)만
  페널티.** 야생종은 대부분 순수 초식(diet 12~30)이고 순수 육식(포식자 85)·잡식 청소부(50)만 중간/육식이라,
  이렇게 두면 야생 초식·포식 생태의 효율이 안 변한다 → 통과기준 밸런스가 **잡식 기준선(내 종 diet 50)에만**
  걸린다(밸런스 이동 최소화). 문턱과도 정렬 — 사냥·채집을 막 시작하는 잡식 끝이 가장 비효율.
- **페널티 폭 0.3 은 대멸종 필터 안전 상한.** 프로브(시드 4 + 필터)로 격자를 쓸었다. 0.3 은 세 필터 다
  또렷이 통과, 0.4 부터 대역병 lo 가 경계(3)로, 0.5 는 한파 hi 가 0 으로 붕괴. 실제 `world.test.ts` 필터
  3종(diet 50 기준선)도 0.3 에서 통과 확인.
- **카드·프리셋 문구는 안 건드렸다.** 「포식/초식 본능」 설명은 방향("육식/초식으로 기운다")만 말해 이미
  정확하다 — 효율 연동으로 그 방향이 이제 실질 이득(사냥/채집 효율 최대)을 갖게 됐을 뿐. hunter 프리셋
  (diet 68)도 이제 사냥 효율 98%·채집 72%라 "사냥꾼" 정체성이 효율로 뒷받침된다(backlog 지적 자연 해소).
- **순수 특화 종 단독 자생 불가는 이 작업과 무관.** 프로브에서 순수 초식(diet 20 단독)·순수 육식(85 단독)이
  거의 멸종하는데, 페널티 0 에서도 동일하다 → 내 변경이 만든 게 아니라 기존 문제(backlog 이월 "순수 육식
  자생력 — 잡식 경유가 정답"). 실제 게임에선 프리셋의 보조 형질(herd 다산·무리 등)이 메꾼다.

## 세션 2 — 순수 육식/초식 자생력 진단

### 요약
식성 효율 연동에 이어 순수 육식/초식이 프리셋 보조 형질 없이 자생하는지 프로브로 진단했다. 순수 초식은
이미 자생 가능(grazeEff 1.0 + herd 다산·무리)하고, 순수 육식은 사냥 수입을 2.3배로 올려도 자생 불가 —
병목이 수입이 아니라 사냥 성공 빈도라 이 게임 구조의 근본 한계임을 확인했다. 코드 변경 없이 진단만 남겼다.

### 진단 (프로브: env-1~4 평균, 3000틱, 내 종 사망 원인)
| 게놈 | pop | peak | 굶주림 | 포식 | 판정 |
|---|---|---|---|---|---|
| 순수초식 diet20 | 1.0 | 21.8 | 13 | 22 | 포식이 병목(채집은 됨) |
| 잡식 diet50 | 17.8 | 29.8 | 46 | 16 | 번성 |
| 순수육식 diet85 | 0 | 18.5 | 17 | 0.3 | 굶주림 멸종 |
| 프리셋 herd | 5.5 | 20.5 | 20 | 14 | 자생(다산·무리) |
| 프리셋 hunter(68) | 37 | 44.5 | 65 | 1.5 | 번성(채집 병행) |
| 육식+빠름 diet85 | 0 | 21.8 | 20 | 0 | 굶주림 멸종 |

- 순수 육식 사냥 보너스(huntEff>1.0, `carnivoreHuntBonus`)를 넣어 0~1.3 격자를 쓸었으나 **전 구간 pop 0**
  → 사냥 수입은 병목이 아니었다. 되돌렸다(sim 코드 원복, 진단만 남김).

### 결정 사항
- **순수 육식 자생은 억지로 만들지 않았다.** 병목은 사냥 성공 빈도 + 플레이어 이주 부재(야생 포식자조차
  이주로 연명한다). 풀려면 야생 사냥 생태 대공사(먹잇감 도망/밀도)라 리스크가 크다. hunter 프리셋(68)이
  번성하는 게 "육식 빌드"의 정답. 상세: known_issues "순수 육식 자생은 사냥 수입 상향으로 안 풀린다".
- 「포식 본능」 카드(diet+22)를 잡식(50)에 쓰면 72 → 순수 육식(함정)이 되는 문제는 남아 있다. 카드를 diet+18
  (→68, 채집 병행)로 낮출지, 순수 육식을 극단으로 둘지, 야생 생태를 손볼지는 사용자 결정 대기.

### 미해결 / 다음 할 일
- 「포식 본능」 함정 회피(카드 diet 조정) vs 순수 육식 살리기(야생 대공사) — 방향 결정 필요.
  → 세션 3 에서 "여러 테크로 살린다"로 결정(사용자).

## 세션 3 — 식성 테크 다양화: 질주형 육식(사냥 스퍼트)

### 요약
사용자 방향("여러 포식자가 다 가능하게 — 잡식이 여러 테크이듯 육식·초식도 다양한 테크를 탈 수 있게")에
따라, 순수 육식을 살리는 첫 축으로 **질주형**(speed 기반 사냥 스퍼트)을 구현했다. 실제 자연의 순수 육식
생존(소수·큰 먹이·긴 포만·도망 잡는 무기)을 모방하는 로드맵의 첫걸음. 순수 육식이 먹잇감을 추격할 때
속도가 올라 도망치는 초식을 따라잡는다. 프로브로 "질주형 자생 + 잡식 밸런스 보존" 양립점(0.3)을 찾았다.

### 주요 변경
- `src/sim/behavior.ts` — `huntSprintFactor(diet, hunting)` 순수 함수. 순수 육식(diet≥dietGrazeMax)이 추격
  중이면 maxSpeed 가 오른다(문턱 70=0, 완전 육식 100=최대). maxSpeed 계산에 배수 적용.
- `src/sim/params.ts` — `huntSprintBonus`(0.3).
- `src/sim/world.test.ts` — 스퍼트 규칙 단위 테스트 3건. + 그림자 매복자 테스트를 여러 시드(env-1~3)로
  견고화 — 밸런스 변경 시 단일 시드 조우 노이즈로 깨지던 것.
- `docs/ROADMAP.md` — "식성 테크 다양화" 비전(육식 사냥법 5갈래 + 초식 생존법 + 공통 포만) 기록.
- `docs/design/known_issues.md` — smoke 스크립트가 Windows 에서 vite preview 를 orphan 으로 남기는 실수 기록.

### 측정 (프로브: 시드 6, 3000틱)
| sprint | 잡식기본 | 극단질주(100/sp90) | 한파 hi |
|---|---|---|---|
| 0 | 14.7 | 1(멸종직전) | 8 |
| **0.3** | 14.7(무영향) | 3(자생) | 9 |
| 0.4 | 6.8(급락) | 3.2 | 7 |
| 0.5 | 7.3 | 5.3 | 10 |

- 0.5 는 극단 질주형이 더 잘 살지만(5.3) 야생 포식자가 스퍼트로 잡식 내 종을 과하게 잡아 잡식이 절반으로
  급락 → 승리 밸런스·game.test 붕괴. 0.3 은 잡식 무영향 + 질주형 겨우 자생(글래스캐논).
- speed 낮은 육식(50)은 스퍼트를 올려도 pop 0 — 도망을 못 잡는다(질주 테크는 speed 가 열쇠).

### 결정 사항
- **스퍼트는 순수 육식 + 추격 중일 때만.** 잡식·초식·야생 초식은 영향 0. 단, 야생 포식자(diet 85)는 받으므로
  "포식자가 무서워지는" 부작용이 있다 — 그게 잡식 밸런스를 흔들지 않는 상한이 0.3.
- **이번엔 스퍼트만(변수 하나씩).** "큰 사냥·긴 포만"(성공 사이 생존)은 로드맵 다음 축. 스퍼트로 성공 빈도를
  올리고, 폰 체감상 질주형이 약하면 포만으로 보강한다.
- **매복자 테스트를 여러 시드로 견고화.** boss members 는 게놈이 없어 스퍼트를 직접 안 받는다 — 스퍼트가
  야생 생태를 미세 이동시켜 env-1 에서 조우가 우연히 0 이 됐다. 단일 시드 이벤트 테스트의 취약성.

### 미해결 / 다음 할 일
- 질주형 육식 폰 체감 — 순수 육식 질주 빌드가 재밌게 자생하는지, 포식자가 무서워진 게 긴장으로 읽히는지.
- 육식 다른 테크(무리사냥=herding, 독니=venom 공격독) + 공통 "큰 사냥·긴 포만" — 로드맵.
- isolation/swarm members 테스트도 같은 단일 시드 취약성 — 다음에 깨지면 stalker 처럼 여러 시드로.

## 세션 4 — 드래프트 데스크톱 UX + 지느러미 육지 사망 버그 + 스탯 스케일 확인

### 요약
사용자 지적 세 건. (1) 데스크톱 드래프트에서 카드 클릭=즉시 선택 + 내 종 팝업을 오른쪽 인라인으로.
(2) 「지느러미」로 육지 종이 물 전용이 돼 뭍에서 굶어 죽는 버그 — 수영 카드 상한으로 수정. (3) 스탯
스케일(내 종 200 / 야생 100)·공격력 무적 여부 확인.

### 주요 변경
- `src/ui/draftPanel.ts` · `panelStyles.ts` — 데스크톱(≥860px): 카드 클릭이 곧 선택(미리보기는 호버·화살표),
  내 종 팝업을 중앙 오버레이(dim) 대신 화면 오른쪽 인라인으로. 모바일은 그대로.
- `src/game/cards.ts` — applyCard 에서 내 종 swimming 을 aquaticOnlyThreshold-1(89)로 막는다. 카드로는
  물 전용이 안 되고 수륙양용까지만.
- `src/game/cards.test.ts` — 수영 카드 누적이 물 전용 문턱을 못 넘는다 강제.
- `docs/design/known_issues.md` — 지느러미 육지 사망 버그 + smoke taskkill 팁 기록.

### 결정 사항
- **물 전용은 카드로 못 되게(수륙양용 상한).** 사용자 방향: "수륙양용이되 바다까지 들어가게만". 진짜 물
  전용(바다 거주)은 야생 물고기 떼만. TRAIT_CEILING 낮추기는 야생 물고기(95)까지 잘라 안 됨 → applyCard 에서만.
- **스탯 스케일(질문 답).** 내 종만 speed·vision·attack·fertility·herding 200, 야생 100 은 의도(내 종=카드로
  키우는 성장 대상). 야생 100 상한은 실질 무의미(야생은 카드 없어 100 근처도 안 감).
- **공격력 무적(질문 답).** 공격력이 상대보다 35 이상 높으면 그 상대는 못 문다(biteIgnoreDiff) — 포식엔
  사실상 무적. 단 굶주림·환경·대멸종·노화·독은 그대로라 완전 무적 아님(공격력 몰빵 시 다른 형질 취약).

### 미해결 / 다음 할 일
- 데스크톱 팝업 인라인 위치(오른쪽 상단) 실기 조정 여지 — 카드와 겹치는지 데스크톱 확인.
