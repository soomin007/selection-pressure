# 2026-07-04 세션 로그

## 요약
ROADMAP "야생 진화 심화"의 다음 단계 착수·완료 — 야생종이 자기 무리가 겪는 **압력별로** 적응한다.
기존엔 추위→고대사 하나뿐이었는데, 먹이 부족→저대사(효율), 포식자 노출→속도·무리(빠르고 뭉쳐 산다)를
더했다. 대사는 추위(위)와 먹이 부족(아래)이 밀당한다. 그리고 진화가 **폰에서 보이게** — 야생 텍스처
키에 거친 게놈 서명을 붙여, 형질이 크게 움직이면 스프라이트가 따라 바뀐다(전엔 종 id 고정이라 진화가
겉모습에 안 나타남). 99 테스트 통과(+4), 통과기준 밸런스 보존.

## 주요 변경
- `sim/world.ts`: `maybeEvolveWild` 개편 — 무리별로 추위·평균 에너지·포식자 노출 비율을 한 번에 측정.
  적응 로직을 순수 함수 `adaptWildTraits(traits, WildPressure)`로 분리(export, rng 미사용=결정론=단위
  테스트 쉬움). 미세 드리프트는 호출부에 그대로(독립 rng). `areFriends`·`Traits` import 추가.
- `sim/params.ts`: 야생 진화 파라미터 추가 — wildMetaBase 30·wildColdMetaGain 60(기존 "30+추위×60"
  보존)·wildScarcityEnergyThreshold 0.35·wildScarcityMetaDrop 40·wildPredSpeedRange 40·wildPredHerdRange 40.
- `render/worldView.ts`: 야생 텍스처 키 `"s"+id` → `"s"+id+":"+wildGenomeSignature`. 새 함수는 형질을
  12 단위 버킷으로 뭉뚱그려(미세 드리프트엔 안 바뀌고 압력 적응 같은 큰 변화만 새 텍스처 → 캐시 상한).
- `sim/world.test.ts`(+4): adaptWildTraits 순수 테스트 4개 — 포식→속도·무리↑, 포식 없으면 불변,
  먹이부족→저대사, 추위와 먹이부족의 대사 밀당.

## 결정/기록
- **적응 로직을 순수 함수로 분리한 이유**: 방향성(포식→속도↑ 등)을 결정론적으로 정확히 검증하려면
  라이브 시뮬(사냥·이동 간섭, 노이즈)이 아니라 압력 입력을 직접 준 단위 테스트가 필요. 측정(world)·
  적응(순수)·드리프트(rng)를 3분리해 각각 견고. known_issues의 "메커니즘 작동으로 테스트" 원칙 실천.
- **밸런스 보존 설계**: (1) 굶주림 끌기는 평균 에너지 < 0.35 에서만 켜져 배부른 시련 상황은 기존 대사식
  그대로. (2) 포식 적응은 목표=현재+노출비율×범위를 wildAdaptRate(0.05)로 천천히 따라가 짧은 시련
  (750~2000틱=2~6주기)엔 몇 점만, 긴 런(6000틱=20주기)엔 10~30점 이동. 99 테스트 통과가 이를 확인.
- **프로브로 강도 확정**(4시드×6000틱): 피식자(작은 풀벌레 spd 37→55·hd 75→88, 물고기 떼 46→62·53→76)는
  빨라지고 뭉치고, 최상위 포식자는 노출 적어 덜 오르고, 포식 없는 무리(느린 거북 20→19)는 안 오른다
  (다양성 유지). 추운 s3 는 대사 전반↑(40→66), 따뜻한 env-1 은 유지/하락. pop 37~65 건강(폭발·멸종 없음).
- **진화를 보이게 = 텍스처 갱신이 필수**: 야생은 공유 게놈이 진화하는데 텍스처가 종 id 로 고정이라 여태
  겉모습이 안 변했다(진화가 화면에 없음 = visible-feedback 위배). 거친 12-버킷 서명으로 "눈에 띄는 변화만"
  새 스프라이트 → 캐시 폭증 없이 진화가 폰에 드러난다.

## 세션 2 — 독 중독 표식 + 물고기 떼 실체화 (폰 피드백)

### 요약
폰 피드백 2건. (1) "야생 진화가 체감 안 됨 — 물고기가 너무 적어서 무리·진화가 안 보임": 물고기 떼를
독립 rng 로 보강(시작 ≈15 학교) + **깊은 바다 먹이**(물 전용 종만 먹는 전용 니치)로 바다 풀뜯이와
경쟁을 분리해 학교가 유지되게. (2) "독먹이를 삼켜 중독됐을 때 겉으로 표시 나면 좋겠다": tint 곱셈만으론
초록 생물이 탁해질 뿐이라, 종 불문 **보라 오라 + 피어오르는 독 방울**을 추가. 103 테스트 통과(+4).

### 주요 변경
- **독 중독 표식** (`render/worldView.ts`): poison>0 개체에 playerG(스프라이트 아래)로 맥동 보라 오라 +
  회전하는 독 방울 3개. 독 세기(poison/venomOnHit)로 크기·굵기 스케일. tint 는 유지(부차).
- **물고기 실체화**: `sim/food.ts` deep 플래그(aquatic/mountainous 형제). `sim/params.ts` deepFoodPatches
  65·seaHerdPad 10(seaFoodPatches 는 55 원복 — 바다 풀뜯이 밸런스 보존). `sim/world.ts` spawnDeepFood
  (독립 rng `-deepfood`) + spawnWildHerdPadding(독립 rng `-herdpad`, 물 전용 종만). `sim/behavior.ts`
  nearestFood 에 deep 분기(swimming≥aquaticOnlyThreshold=물고기만). worldView 깊은 바다색(진한 남청).
- `sim/deepFood.test.ts`(신규 4): 물 타일 배치·플래그 비겹침·결정론·물고기 학교 스폰 규모.

### 결정/기록
- **물고기 떼는 "떼여야 관찰이 된다"**: 소수 개체 게임이라도 야생 "떼종"이 5마리면 무리·진화가 안 읽힌다.
  내 종은 소수 유지하되 야생 떼종만 시각적으로 떼답게. 개체 수는 독립 rng 스폰(친척·바다먹이 패턴)이라
  메인 스트림 불변 → 육지 통과기준 밸런스 보존(103 테스트 통과).
- **경쟁 배제는 자원 분할로 푼다(known_issues 재확인)**: 얕은 바다 먹이만 늘리면(75·110·150 스윕) 바다
  풀뜯이(양용)가 더 빨리 먹어 물고기는 여전히 못 큼(grazer 41까지 폭증). 깊은 바다 먹이=물고기 전용
  니치로 경쟁을 끊어야 학교가 산다. mountainFood(날개 전용)의 물속 대칭.
- **붐-버스트는 없애지 않고 받아들인다**: 물고기는 다산형(fertility 70)이라 먹이를 늘려도 평균만 오르고
  (avg 6→12) 바닥은 낮게 출렁인다(min 1~6). 이건 소비-자원 진동의 본질 = 살아있는 생태. 시작은 항상
  15 학교, 이후 10~20으로 커졌다 작아졌다. 억지 안정화(fertility↓)는 종 정체성 훼손이라 안 함.
- **areaScale 함정 재현**: 패딩 수에 areaScale 을 곱했다가 "개체 수는 절대(맵 무관)" 테스트가 깨짐
  (big 159 vs small 79). areaScale 은 먹이·상한 밀도에만 — 개체 수 절대 원칙 재확인.

## 세션 3 — 먹이 목표 합리화(히스테리시스) (폰 피드백)

### 요약
폰 피드백: "생물이 눈앞에 더 쉬운(가까운) 먹이가 있어도 원래 쫓던 것만 계속 쫓는다. 이게 맞나?" +
"육지 생물이 물가에서 끼인다". 진단: chooseGoal 의 목표 고정(commit)이 과했다 — 기존 목표가 감지
범위 안이면 더 가까운 먹이와 비교조차 없이 그것만 반환. 떨림(목표 진동) 방지로 넣은 게 지나쳐 불합리한
고집 + 물가 먹이 고집→끼임의 원인. 해결: 히스테리시스로 "훨씬 가까운 먹이면 갈아탐". 103 테스트 통과.

### 주요 변경
- `sim/behavior.ts` chooseGoal 재구성: 매 틱 가장 가까운 후보(먹이·먹잇감)를 먼저 살핀 뒤, 기존 목표는
  `cand2 < cur2 × targetSwitchGain` 일 때만 갈아탄다(아니면 유지). 목표에 다가갈수록 cur2 가 줄어 더
  끈질겨짐(합리적). `sim/params.ts` targetSwitchGain 0.5(새 먹이가 기존 거리의 ≈71% 안이어야 전환).

### 결정/기록
- **떨림 방지와 합리성의 균형**: 예전 고정은 "감지 범위 안이면 무조건 유지"라 떨림은 막았지만 눈앞의
  쉬운 먹이를 무시. 히스테리시스(0.5)는 조금 가까운 정도엔 안 바꿔(떨림 유지) 훨씬 가까우면 바꿔(고집
  제거). 프로브: 오래 막힌(stuck>25) 개체 평균 0.2~0.5마리(거의 없음) — 물가 끼임도 크게 완화.
- **밸런스 보존 확인**: 목표 선택은 채집 효율에 직결(known_issues: sim 행동 바꾸면 개체수 스케일 이동)이나
  103 테스트 전부 통과 + 프로브 pop 63~78 건강. 매 틱 후보 탐색은 소수 개체라 성능 무관.

### 미해결 / 다음 할 일
- **폰 검증**: 생물이 눈앞의 가까운 먹이로 갈아타는지(원래 목표 고집 사라졌는지), 물가 끼임이 줄었는지.
  너무 자주 갈아타 산만하면 targetSwitchGain↓(더 끈질기게).
- 남은 폰 피드백: 야생 초식 개체 수 늘리기(먹이사슬 하위일수록 많이) / 독 중독 목격 빈도(포식자가 내
  독 종을 삼키는 장면을 잘 못 봄 — 관찰 빈도·표식 지속).
