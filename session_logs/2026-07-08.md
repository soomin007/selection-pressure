# 2026-07-08 세션 로그

## 요약
docs 핸드오프(방향 3a "포근한 관찰")의 디자인 토큰·타이포·형태 언어를 DOM 오버레이 UI 전반에 적용.
차가운 남색 계열에서 따뜻한 흙빛 유리 톤으로 전환하고, 형질 6색 매핑을 스탯바·범례·보고서에 공유.
헤드리스 부팅·스크린샷 9종으로 시각 검증 완료.

- `index.html` — Google Fonts(Black Han Sans·Jua·IBM Plex Sans KR·JetBrains Mono) 로드 + 배경/theme-color 따뜻하게
- `src/ui/panelStyles.ts` — :root 디자인 토큰(색·활자·반경) 전면 선언 + 공유 클래스(하단 시트·카드·입체 키 버튼·로비·컨트롤 알약·멈춤 메뉴) 재작성
- `src/ui/traitDisplay.ts` — 형질 6색 매핑 `TRAIT_COLORS`/`traitColor()` 추가(속도 amber·시야 blue·공격 red·번식 lime·무리 purple·대사 orange)
- `src/ui/draftPanel.ts` — 카드 왼쪽 액센트·델타 색을 대표 형질 색으로(`dominantTrait`)
- `src/ui/resultPanel.ts` — 사망 원인 색 3a 팔레트, 버튼 입체 키/보조로 정리
- `src/ui/creatureCard.ts`·`buildPanel.ts`·`presetPanel.ts` — 따뜻한 유리 + 형질 6색 스탯바, 시작 버튼 lime 입체 키
- `src/ui/levelUpScreen.ts`·`momentOverlay.ts` — 진척도/멸종 순간 배경·큰 글자(Black Han Sans) 따뜻하게
- `src/ui/runReportScreen.ts` — 형질 흐름 6색 정렬, 사건 색·배경 3a 톤
- `src/ui/glossary.ts` — 대백과 크롬(패널·헤더·칩·표) 따뜻하게

## 결정 사항
- 스펙이 "DOM 오버레이 레이어만 규정"한다고 명시 → Pixi 렌더(hud/minimap/worldView)와 sim/game 은 건드리지 않음.
- 토큰은 `panelStyles.ts`의 `:root`에 한 번 선언, 인라인 스타일은 `var(--*)` 참조. 토큰을 안 쓰던 컴포넌트엔 `ensurePanelStyles()` 호출을 추가해 부팅 순서와 무관하게 토큰 보장.
- `traitPanel.ts`는 main.ts 에 배선 안 된 죽은 dev 도구라 손대지 않음.
- 목업 번들(각 10MB+)은 커밋하지 않고 gitignore — 스펙은 `docs/UI_HANDOFF.md`가 단일 진실.

## 세션 3 — HUD/타임라인/범례 DOM 이관(포맷 통일 마무리)
### 요약
직전 세션에서 "Pixi라서 별도 큰 작업"이라며 남겨둔 상단 HUD·타임라인·범례를 DOM 으로 옮겼다.
핸드오프의 취지(미리 포맷을 통일해 큰 후속 작업을 없앤다)를 어긴 것이라는 지적을 받고 바로 진행.
좌하단 조작 버튼·미니맵·위협 배너까지 3a 톤으로 맞춰 남은 시각 불일치를 없앴다.

### 주요 변경
- `src/ui/hudPanel.ts`(신규) — 정보 카드(내 종/야생·시대·단계·남은 시간·낮밤·레벨 진행바·사망 알림,
  데스크톱 추이선) + 진행 타임라인(보스 red·멸종 ink 마커) + 접이식 "종 안내" 범례를 DOM 으로.
- `src/render/hud.ts` 삭제 — Pixi HUD 를 hudPanel(DOM)이 대체.
- `src/main.ts` — hud 배선을 `createHudPanel()`+`hud.update({...})`로 교체(라인 11·65·72·480).
  좌하단 "한 마리 관찰"을 lime 입체 키, 줌 +/− 를 계측 알약으로.
- `src/render/minimap.ts` — 프레임/점 색을 3a 토큰(흙빛 프레임, lime 무리, red 보스).
- `src/render/threatBanner.ts` — 위협 예고 색을 amber/red 토큰으로.

### 결정 사항
- HUD 를 Pixi 유지+색만 바꾸는 대신 DOM 으로 이관: 커스텀 폰트(Jua·JetBrains Mono)를 브라우저가
  안정적으로 처리하고, 목업이 HUD 를 DOM 알약 카드로 규정하며, 두 렌더 시스템의 시각 격차를 없애기 위함.
- 미니맵은 지형 픽셀 맵을 그리는 실제 렌더라 DOM 이관 무의미 → 프레임 색만 정렬(Pixi 유지).
- dev 패널(?dev)은 이미 호박톤이고 플레이어 화면 아님 → 손대지 않음.

### 미해결 / 다음 할 일
- 폰 실기기에서 HUD 폰트 로딩(FOUT)과 타임라인 마커 라벨 겹침(마커가 매우 가까울 때) 눈으로 확인.
- 개체 초상 카드 실제 스프라이트 썸네일(오프스크린 렌더)은 여전히 색 점 — 별도 논의.

## 세션 4 — 내부 폴리시: 개체 카드 실제 스프라이트 썸네일
### 요약
외부 디자인 수령을 당분간 중단(사용자 결정)하고 내부 폴리시로 전환. 개체 관찰 카드가 색 점만 쓰던 것을
실제 생물 스프라이트 썸네일로 교체 — 개체별 게놈으로 오프스크린 렌더해 "이 아이의 생김새"가 카드에 선다.

### 주요 변경
- `src/ui/creatureCard.ts` — 헤더의 색 점을 56×52 둥근 초상 상자로. `makeCreatureTexture`+`extract.canvas`
  (프리셋 창과 같은 패턴)로 개체 게놈 초상을 렌더. 선택이 바뀔 때만(id 변화) 다시 그리고 텍스처 즉시 해제.
  `CreatureCardData` 에 `id`·`genome` 추가, `createCreatureCard(renderer, cb)` 로 renderer 주입.
- `src/main.ts` — `createCreatureCard(app.renderer, …)` 배선, update 데이터에 `id`·`genome` 전달.

### 결정 사항
- 초상은 매 프레임이 아니라 개체 선택 변화 시에만 렌더(GPU 읽기 비용) — id 캐시. 종횡비 유지(프리셋 창 방식).
- 핸드오프 문서(UI_HANDOFF.md)는 외부 라운드 예정 없어 이번엔 안 건드림 — 내부 폴리시에 집중.

### 미해결 / 다음 할 일
- 폰 실기기에서 초상 썸네일 선명도(작은 캔버스 스케일) 확인.
- 남은 관찰성 폴리시 후보: 신생아 표식(잠깐 빛나는 고리), 개체 즐겨찾기/이름 고정, 무리 형질 분포 표시(§9).
