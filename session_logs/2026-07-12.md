# 2026-07-12 세션 로그

## 요약
식성 테크 다양화의 다음 축 — 순수 육식 "큰 사냥·긴 포만"을 구현했다. 질주 스퍼트(세션 3)가 사냥 성공
"빈도"를 올렸다면, 포만은 성공 "사이 생존"을 늘린다. 순수 육식은 한 사냥으로 크게 먹고(gorge) 100 위로
에너지를 비축해(reserve) 대사로 천천히 태우며 다음 사냥까지 버틴다(실제 대형 포식자). 이로써 로드맵의
"질주+포만 = 순수 육식 자생 최소 조합"이 완성됐다. 프로브로 야생 밸런스·대멸종 필터 보존 상한을 튠했다.

## 주요 변경
- `src/sim/behavior.ts` — `carnivory01(diet)` 헬퍼 추출(문턱 70=0·완전 육식 100=1). 기존 `huntSprintFactor`
  가 이 헬퍼를 쓰게 리팩터(스퍼트와 공유하는 "순수 육식일수록" 스케일). `gorgeFactor(diet)`("큰 사냥"
  사냥 수입 배수)·`maxEnergyFor(diet)`("긴 포만" 에너지 상한) 순수 함수 추가. `devour` 가 둘을 적용.
- `src/sim/params.ts` — `carnGorgeBonus`(1.0)·`carnGorgeReserve`(60) 추가. 짝 메커니즘 이유를 주석에.
- `src/sim/world.test.ts` — carnivory01·gorgeFactor·maxEnergyFor 규칙 테스트 3건(무영향·최대·단조).
- `docs/ROADMAP.md` — 식성 테크 착수 순서 갱신(질주+포만 완료 → 남은 축: 무리사냥 우선).

## 결정 사항
- **큰 사냥과 긴 포만은 한 짝이다(둘 다 필요).** 큰 사냥만 있고 창고(상한)가 100이면 넘치는 영양이
  버려져 무의미하다 — 세션 2가 사냥 수입만 2.3배 올려 실패한 진짜 이유(상한에 막혔다). 창고만 키우고
  사냥이 작으면 드문 사냥으로 못 채운다. 그래서 gorge(+상한 초과분)와 reserve(상한 자체)를 함께 뒀다.
  "변수 하나씩" 원칙 위반이 아니라 한 메커니즘의 결합 파라미터(프로브도 함께 쓸었다).
- **carnivory01 로 스케일 → 잡식(diet 50) 완전 무영향.** carn01=0이라 gorgeFactor 1·상한 100 그대로 →
  통과기준(잡식 기준선)·야생 초식(diet 12~30) 밸런스 불변. 야생 포식자(diet 85, carn01≈0.5)만 절반
  세기를 받으므로 폭주 안 하는 상한을 프로브로 튠(스퍼트 0.3과 같은 방식).
- **값 B=1.0 R=60 채택.** 순수 육식 한 사냥 = 36×2.0=72(큰 사냥), 상한 160으로 비축(긴 포만). 8시드
  프로브: 스퍼트만이면 순수 육식 0/8 멸종(피크 22.8→붕괴) → 포만 얹으면 4/8 자생(글래스캐논). 깔끔한
  라운드 값이면서 필터·sanity를 다 보존하는 지점.
- **잡식 "최종" 개체 수는 노이즈 지배 → 피크로 판단.** 같은 B=1 R=60이 4시드 프로브 13.5 vs 8시드 8.9로
  요동친다(카오스 생태). 하지만 잡식 피크는 전 구간 24~25로 안정 → 조기 붕괴 없음(승리 밸런스 안전).
  포식자는 5→7로 온건 상승. 최종값으로 튜닝하면 노이즈에 휘둘린다는 교훈(known_issues 계열).
- **렌더 손상 없음.** 에너지바(`creatureCard.ts`)가 `Math.min(1,…)`로 클램프해 포만 개체는 가득 찬 바로
  정상 표시(오히려 옳은 피드백). sim 변경이 완전 자족적이라 UI 후속 작업 불필요.

## 세션 2 — 무리사냥(먹이 나눔): herding 을 육식 생존 레버로

### 요약
식성 테크 세 번째 육식 사냥법. 순수 육식이 사냥에 성공하면 사냥감 주위 같은 종 무리가 카커스 몫을 나눠
받는다(늑대가 함께 먹는다). 이 나눔이 herding 을 뒤집는다 — 원래 cohesion 뭉침은 먹이 탐색 범위를 좁혀
육식엔 부채였는데(프로브: herd90 육식이 herd10 보다 못 삼), 나눔이 있으면 뭉친 팩이 소수의 사냥으로 다
같이 먹어 자생한다. speed(잡기)+herding(나눔) 빌드로, 질주+포만(잡고 비축)과 구별되는 축.

### 주요 변경
- `src/sim/behavior.ts` — `packHerdFactor(herding)`(herding 임계 55 기반 선형)·`packShareGain(huntGain,
  diet, herding)` 순수 함수. `devour` 가 사냥 성공 시 사냥감 주위 같은 종 순수 육식 무리에게 몫 지급.
- `src/sim/spatialGrid.ts` — `forEachMatching(x,y,r,fn)`(반경 내 개체에 콜백 — 나눔 지급용).
- `src/sim/params.ts` — `packShareThreshold`(55)·`packSharePerMember`(0.7)·`packShareRadius`(55).
- `src/sim/world.test.ts` — packHerdFactor·packShareGain 규칙 5건.
- `docs/ROADMAP.md` — 무리사냥 완료 표시, 남은 축(사수·독니·완력)으로 갱신.

### 결정 사항
- **"협동 몰이(즉사 확률↑)" 대신 "먹이 나눔(카커스 공유)"으로.** 처음엔 먹잇감 주위 무리 수로 즉사 확률을
  올리는 협동 몰이로 구현했으나, 프로브에서 herding 육식이 baseline 부채(cohesion 탐색 손실)라 즉사 보너스가
  그 손실을 못 넘어 무승부(노이즈). 늑대 무리의 본질은 "함께 쓰러뜨림"보다 "함께 먹음" — 먹이 나눔이 뭉침을
  자산으로 뒤집어 herding 이 진짜 생존 레버가 된다(자생 문제=성공 사이 생존을 직접 푼다). 협동 몰이 코드
  (countMatching·killChance 가산)는 버리고 나눔(forEachMatching·devour 지급)으로 교체.
- **herding 스케일은 임계(55)로 — 제곱·세제곱은 야생 격리 실패.** 나눔이 야생 포식자(diet 85·herding 40)에도
  새면 pop 이 늘 때 뭉쳐 나눠 먹는 되먹임으로 폭주해 잡식 승률이 떨어졌다(game.test "시대 보상 재현"이
  60시드 내 승리 0 으로 회귀). herding² (0.16)·herding³ (0.06) 로도 P 를 올리면 다시 샜다. **임계 55 로 딱
  끊으니** 야생 포식자(40)·잡식·herd10 빌드가 전부 나눔 0 → 잡식 8.9·포식자 7.1·대멸종 필터가 전 P 에서
  바이트 단위 동일(완벽 격리). 순수 육식+herding>55 는 플레이어 무리 빌드뿐(다른 야생 고herding 종
  우림새떼 66·물고기 60 은 다 초식 carnivory01=0 으로 애초에 제외).
- **P=0.7 채택.** 무리herd90 이 0.5→7.0(6/8) 로 뚜렷이 자생(폰에서 잘 읽히는 큰 변화). 격리가 완벽해
  밸런스 리스크 0 이라 값은 순전히 "무리 빌드가 얼마나 센가" 취향 — 폰 체감으로 조정 가능(다른 어떤 것도
  안 흔든다). 나눔은 "잡은 뒤"만 도와 speed 낮은 팩(몰이 sp75)은 못 산다 → speed 는 여전히 필요.

|---|---|---|---|---|---|
| 0 | 0.5 (2/8) | 2.5 | 8.9 | 7.1 | 7/0 |
| **0.7** | **7.0 (6/8)** | 2.5 | 8.9 | 7.1 | 7/0 |

## 세션 3 — 해금 사다리 열람 공간 + 결과 화면 꾸미기(사용자 요청)

### 요약
사용자 지적 두 건. (1) 플레이어 레벨마다 무엇이 열리는지 볼 공간이 없었다(levelUpScreen 은 레벨업
순간에만 잠깐 보여줌). (2) 꾸미기를 바꾸려면 "런 시작 → 설정 → 로비 → 적용 → 재시작" 우회가 강제됐다
(꾸미기 UI 가 로비에만 있고, 결과 화면 "다시 시작"은 로비를 안 거쳐 바로 새 런으로 감).

### 주요 변경
- `src/ui/unlockLadder.ts`(신규) — 레벨별 해금(UNLOCK_TIERS) 열람 오버레이. 현재 레벨·진척도 바 +
  사다리(열림/다음/잠김 레벨N 상태). 대백과처럼 별도 오버레이, 열 때마다 현재 레벨 재조회.
- `src/ui/cosmeticPicker.ts`(신규) — 로비 인라인 꾸밈 칩을 재사용 컴포넌트로 추출(el + refresh).
- `src/ui/lobby.ts` — cosmeticPicker 사용 + 대백과 옆 "해금" 버튼(onLadder). 동작 동일.
- `src/ui/resultPanel.ts` — "다음 판 준비" 구획(cosmeticPicker + "해금 사다리 보기"). show 에서 refresh.
- `src/main.ts` — applyCosmetics·unlockLadder 를 결과 패널 생성보다 먼저 선언(콜백으로 넘기므로 TDZ
  방지, known_issues). result onNewRun/onContinue 가 applyCosmetics 호출.

### 결정 사항
- **꾸밈 우회 제거 = 결과 화면에 픽커를 붙임(로비→결과 리다이렉트 아님).** "다시 시작"을 로비로 되돌리면
  빠른 재시작 흐름에 매 판 클릭이 는다. 대신 결과 화면 자체에 꾸밈 픽커를 둬 우회 없이 바로 바꾸게 했다.
- **픽커를 재사용 컴포넌트로.** 로비·결과 두 곳이 같은 컴포넌트를 써 꾸밈 UI 가 어긋나지 않는다
  (known_issues "한 효과가 여러 표면에 걸치면 함께 갱신"의 실천 — 아예 한 소스로 합쳤다).
- **해금 사다리는 상시 열람(레벨업 순간과 별개).** UNLOCK_TIERS 를 현재 레벨 기준으로 열림/다음/잠김
  으로 펼쳐 "다음에 뭐가 열리지?"를 언제든 본다. 꾸밈(업적 기반)은 여기 안 넣음 — 레벨 해금과 다른 축.
