## 결정 사항
- **색은 서로·대멸종 틴트와 겹치지 않게 벌렸다.** 한파(파랑)·폭염(주황)·역병/풍요(녹황·황갈)와
  시련 색이 헷갈리지 않도록 선택. 대멸종과 시련은 동시에 안 뜨고 우선순위(대멸종 분기가 먼저)라
  겹쳐도 대멸종이 이기지만, 그래도 시각적으로 갈라 둠.
- **색뿐 아니라 맥동(속도·진폭)으로 질감까지 구분.** 우글거림(빠름)·파상 공격(큰 진폭)·냉혹한 고립
  (느림)·그림자(은은). 폰에서 색이 가장 강한 신호라 색만으로도 갈리지만 질감을 더해 성격을 실었다.
- **타임라인 마커 색 세분화는 이번 범위 밖.** game.timeline 이 넘기는 marker.kind 는 "boss"/
  "extinction" 뿐이고 시련 종류는 rng pick 이라 예고 시점엔 미정. 예고는 이미 텍스트(종류+힌트)로
  알리므로, 이번 핵심은 "시련 진행 중 화면"을 종류별로 구분하는 데 집중.
- **시각=로직 1:1 원칙 준수(known_issues).** 전역 효과라 화면 전체 틴트로만 표현(국소 점 없음).
  세션 10 보스 개편의 자연스러운 후속 — 그때 "전역=화면 전체 안개"로 통일한 걸 종류별로 세분화.

## 세션 2 — 대멸종 종류 예고

### 요약
backlog "바로 후보"의 대멸종 종류 예고를 구현했다. 그동안 대멸종은 종류가 실행 시점 rng.pick 이라
예고 getter(부작용 금지)에서 종류를 못 정해 "곧 대멸종"까지만 떴다. 보스 큐 패턴을 대칭 적용해
종류를 미리 정해 저장하니, 이제 "곧 혹독한 추위!" 처럼 종류명 + 대응 힌트를 미리 알린다. 81 통과(+2).

### 주요 변경 (3b411c2)
- `src/game/game.ts`:
  - `extinctionQueue` 필드 + 전용 독립 rng `extRng`(`${seed}-ext`). setupRun 에서 `shuffle`로 큐 생성.
  - `beginStage` 대멸종 분기: `stageRng.pick` → `extinctionQueue.shift()`(예고 peek 과 같은 값).
  - `upcomingThreat` 대멸종 분기: `extinctionQueue[0]` peek 로 `곧 <종류명>!` + `extinctionCounter` 부제.
  - `extinctionCounter(et)` 신설(보스 bossCounter 대칭, 짧은 대응 힌트). `shuffle` 제네릭화(보스/대멸종 공용).
- `src/game/game.test.ts`(신규): 대멸종 예고 == 실제 대멸종 종류 end-to-end 검증(런을 headless 로
  끝까지 돌려 관찰) + 같은 시드 재현(결정론).

### 결정/기록
- **독립 rng 스트림(-ext)으로 stageRng 순서 보존.** 대멸종 종류 pick 을 stageRng 에서 빼 별도
  스트림으로 옮겨, 보스 순서(stageRng shuffle)와 시드 재현을 1비트도 안 건드림(known_issues "독립 rng").
  대멸종 종류 자체는 시드 고정 시 재현되되, 기존 보스/밸런스 테스트는 무영향.
- **보스와 완전 대칭(peek == shift).** 보스가 이미 이 패턴으로 검증됐고(세션 10), 대멸종도 같은
  큐 구조라 "예고한 것 == 닥치는 것"이 자명하게 보장. 예고 문구 구조({title, sub})도 동일해 main
  배선(threatBanner)을 안 건드림.
- **game end-to-end 테스트 도입.** Game 이 순수 TS(Pixi 무관)라 headless 로 런을 완주시켜 예고-실제
  일치를 직접 관찰. backlog 목적("예고가 정확해야")을 그대로 검증하는 게 가장 정직. 통과기준 낮아
  대부분 완주, 도중 멸종 시드는 건너뛰고 win 런에서만 검증.
- **SCHEDULE 상 대멸종은 1개(피날레)**지만 큐는 4종 셔플로 채워 향후 대멸종 다수화에도 서로 다른
  종류가 나오게 대비(보스 큐와 동일).
