## 세션 — 생물 떨림 끝장 조사 + 사건 연출 (remote-control)

### 요약
폰에서 떨림이 계속된다는 반복 피드백을 끝까지 추적했다. 회전→위치→배회→오버슈트→무리→사냥까지
원인을 차례로 잡았으나, 결국 임시 프로브(종별 reversal% 측정)로 "전문종 떨림은 적은 개체가 드문
전문 먹이를 먹는 생태의 본질이라 sim 으로 못 없앤다"를 객관 확인. 렌더 평활까지 넣었지만 사용자
체감 부족 → 사용자 결정으로 떨림은 접고 콘텐츠로 전환. 추천 작업으로 "사건 연출"을 구현했다.

### 주요 변경 (떨림)
- `worldView.ts` — 회전 이징 프레임률 독립화 + 회전 데드존(방향 굳히기) + 위치 저역통과 평활(dispPos).
- `behavior.ts`/`entity.ts`/`params.ts` — 배회 헤딩 보존(wanderAngle), 도착 감속(arrive/toward),
  사냥 도착 감속, cruiseFactor↓.
- `params.ts` — 무리 cohesion comfort zone(격자 COM 양자화 점프 추적 중단).
- `debug.ts` — 폰 진단·튜닝 URL 토글(?norot/?nointerp/?showalpha/?dz/?rotk/?smooth) + 좌하단 HTML 배지.

### 주요 변경 (사건 연출 — 콘텐츠 전환)
- `world.ts` — `emit(kind,x,y)` + `events: VisualEvent[]`(전 종, 위치 포함, rng 미사용=결정론 무관).
- `behavior.ts`/`boss.ts` — 탄생/자연사/잡아먹힘/보스처치 지점에서 emit.
- `render/effects.ts`(신규) — 사건별 짧은 연출(탄생=초록 링, 자연사=회색 스러짐, 잡아먹힘=빨강 터짐).
- `main.ts` — 매 프레임 world.events → effects, 런 바뀌면 clear.

### 결정/기록
- **떨림은 sim 파라미터로 못 잡는다(프로브로 막다른 길 확인). 반복 금지** — 자세한 건 known_issues.
  전문종 reversal ~2~3% vs 내 종 0.5%. 먹이 촘촘화는 더 자주 먹어 역효과.
- **사용자 판단으로 떨림 추격 중단, 콘텐츠로 만회 전략 전환.** 추천=사건 연출(가독성+생동감).
- **시뮬/렌더 분리 유지:** sim 은 사건을 사실로만 기록(위치), 렌더가 해석해 그림. 결정론·밸런스 불변.

### 미해결 / 다음
- 폰에서 사건 연출 체감(탄생/죽음/잡아먹힘이 읽히는지, 과하지 않은지). 빈도·크기 튜닝 여지.
- 떨림은 추후 정교한 아트 단계에서 자연 완화 기대. sim 재시도 금지.
