  - `world.species` 색을 읽어 그림. 내 종은 화면 속 초록 고리와 맞춘 테두리로 표시.
  - 종 구성 시그니처(`id:color`)가 바뀔 때만 다시 그림 → 매 프레임 비용 없음(거의 정적).
  - 로비(상태줄 빈 문자열)에선 숨김. 먹이 색 3개는 worldView 의 FOOD_COLORS 와 동기화(상수 복제+주석).
- `ACTIVE_WORK.md`(git 미추적) — 4개 레인 슬롯 + 사용법 추가, 레인 C claim 후 작업.

### 결정 사항
- **범례를 신규 파일+main.ts 마운트 대신 hud.ts 안에 넣음.** 다른 세션이 main.ts 를 점유 중이라
  충돌을 피하려고, 이미 스테이지에 올라가 `world` 를 받는 HUD 에 그렸다. main.ts/worldView 무수정.
- **HUD 는 화면 픽셀 공간(스케일 밖)** 이라 화면 폭을 모름 → 우측 정렬 불가. 기존 HUD 처럼
  좌측 고정 좌표(y≈172)로 배치. `sync(world, statusText)` 시그니처를 안 바꿔(=main 무수정)
  `statusText===""` 로 로비 숨김을 판단.
- **먹이 색은 상수 복제 + 동기화 주석.** worldView 를 안 건드리려고 3개 색을 hud 에 복제.
  DRY 보다 동시 세션 격리를 우선(추후 공용 팔레트로 통합은 backlog 감).
- 커밋은 저장소 관례(한국어 prefix(scope) + bullet, footer 없음) 따름. 내 파일만 경로 스테이징.
