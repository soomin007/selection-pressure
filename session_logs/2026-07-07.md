# 2026-07-07 세션 로그

## 요약
클로드 디자인 협업에 "게임이 어떻게 생겼는지" 전달하려고 주요 화면 8장을 자동 캡처하는 스크립트를
만들었다. 그 과정에서 드러난 모바일 세로 좌하단 UI 겹침(조작 열·개체 정보 카드·드래프트 카드)을
상태별 배타 표시로 수정했다.

## 주요 변경
- `scripts/screenshots.mjs` (신규): vite dev + chromium 으로 로비~드래프트 + 데스크톱 8장 캡처.
  화면 전환을 실제로 구동하므로 부팅 검증도 겸한다(boss-preview.mjs 패턴). 출력은 `screenshots/`
  (각 화면 설명 `README.md` 포함), gitignore 로 제외.
- `src/main.ts`: 좌하단 조작 열(한 마리 관찰·줌)은 "관전 + 개체 미선택"일 때만, 개체 정보 카드는
  "관전 중"일 때만 표시. → 로비·드래프트·개체 선택 상태와 좌하단에서 겹치지 않음(로비에 조작 열이
  떠 있던 문제도 함께 해결).
- `docs/design/known_issues.md`: 좌하단 UI 셋 겹침의 원인·해결·재발 방지책 기록.
- `.gitignore`: `screenshots/` 제외.

## 결정 사항
- 스크린샷 스크립트는 재사용 가치가 있어 커밋(boss-preview.mjs 와 같은 위치·패턴). 출력물만 gitignore.
- UI 겹침은 위치 이동이 아니라 **상태별 배타 표시**로 해결 — 모바일 세로는 폭이 좁아(≤540) 좌우
  분할이 안 되므로, 같은 모서리 위젯은 phase/선택 상태로 시분할한다.
- 데스크톱 08 은 순수 관전을 노렸으나 무리가 커 레벨업이 워낙 빨라 드래프트가 먼저 뜬다 → 가로
  레이아웃 + 형질 패널 + 카드가 한 장에 담기는 이점으로 그대로 채택.

## 미해결 / 다음 할 일
- 이번 UI 수정은 폰 실기 확인 권장 — 겹침이 사라졌는지, 조작 열이 관전 중에만 뜨는지.

## 세션 2 — 런 보고서

### 요약
backlog 최상단의 "런 보고서"를 구현했다. 런이 끝나면 결과 화면에서 "이 혈통의 기록"을 열어 개체 수·형질
추이 그래프와 연대기를 본다. 형질 평균 추이가 개체별 진화를 눈으로 보여준다(오래 묵은 "개체 진화가 폰에서
안 보인다" 문제를 겨냥).

### 주요 변경
- `src/game/game.ts`: RunHistory — 관전 중 1초마다 시계열 샘플(개체 수 + 무리 평균 형질 6종) + 사건 로그
  (시작·카드·보스/시련 넘김·대멸종 견딤·시대 진입·종료). 시대 넘어가도 누적, 새 런에서만 리셋. runHistory getter.
- `src/sim/genome.ts`: `MUTABLE_TRAITS`(변이 6종) export — 보고서 형질 추이 축과 공유.
- `src/ui/runReportScreen.ts`(신규): 개체 수 면적 그래프 + 형질 흐름 꺾은선(50 기준선) + 연대기. self-contained SVG.
- `src/ui/resultPanel.ts`·`src/main.ts`: "이 혈통의 기록 보기" 버튼 + 배선(닫으면 결과 화면 복귀).
- `src/game/game.test.ts`: 히스토리 샘플·사건 + 결정론(같은 시드 재현) 테스트 3종 → 총 137 통과.

### 결정 사항
- 샘플·사건 기록은 game 층에서만(sim rng 미소비) → 결정론·밸런스 무관. game.test 로 재현성 확인.
- 형질 그래프는 개체가 살아있는 샘플만 그린다 — 멸종 순간 평균 0으로 급락하는 건 진화가 아니라 아티팩트라 제외.
- 보고서는 강제로 끼우지 않고 결과 화면의 버튼으로 연다(폰에서 빠른 재시작을 막지 않게).

### 미해결 / 다음 할 일
- 폰에서 보고서 가독성 확인(그래프 선 겹침·연대기 길이) + 승리/정복(여러 시대 누적) 런의 보고서 점검.
- 다음 후보: backlog의 폰 검증·튜닝, 또는 ROADMAP 다음 단계(바이옴별 야생종·청각 감각).
