## 세션 1 — 보스 절차 아트 다듬기(방향 A) + 시각 검증 하니스

### 요약
어제 확정한 보스 아트 방향 A(AI 에셋 보류, 절차 생성으로 간다)를 이행. `render/worldView.ts`의
`drawBossCreature` 도형을 게임 생물(`makeCreatureTexture`) 스타일로 다시 짜 위압적이면서 회전 안전하게
만들고, 5종을 눈으로 확인할 수 있는 프리뷰 하니스를 붙였다. 타입체크·유닛(120)·스모크 통과 + 스크린샷 확인.

### 주요 변경
- `render/worldView.ts`:
  - `drawBossCreature`(WorldView 메서드) → 얇은 래퍼로 두고, 실제 그리기를 모듈 함수 `drawBossShape(g, …)`로
    분리·export(프리뷰 하니스가 같은 함수로 렌더 → 화면과 검증 1:1).
  - 각진 다각형 → **부드러운 곡선 몸통**(quadraticCurveTo). 고정 갈색 `BOSS_OUTLINE` 제거 →
    **색에서 파생한 굵은 단일 윤곽선**(`darken(color,0.42)`)로 통일. **플랫 2톤**(등 하이라이트 `lighten 0.42`).
    **큰 흰자 눈 + 동공 + 반짝**(생물과 통일) + **성난 눈썹**(위압). 흰 송곳니·벌린 아가리(거의 검게) 유지.
  - 종류별: chaser=어뢰형 상어(등지느러미·꼬리·아가리), swarm=통통한 피라니아, raider=앞 뿔 둘+엄니+두 눈,
    isolation=날렵한 늑대(귀·좁은 주둥이), stalker=가시 왕관+번뜩이는 눈 셋(방향성 없는 매복 덩어리).
  - 좌표를 몸 크기 `s` 배수로 정규화한 `P()` 회전 변환 → 진행 방향 회전에도 안 뒤집힘(3방향 스크린샷 확인).
- `boss-preview.html`(신규, 루트, dev 전용·미배포) + `scripts/boss-preview.mjs`(신규): vite dev + chromium 으로
  보스 5종을 3방향 회전으로 렌더해 PNG 저장. `npm run boss-preview [출력경로]` 스크립트 등록.
- `.gitignore`: `boss-preview.png`(기본 출력) 추가.
- `docs/design/boss_art_prompts.md`: 어제 이미 "보류" 결정 노트 반영됨(이번 세션 변경 없음, 참고용 유지).

### 결정/기록
- **보스=절차 생성으로 확정 이행.** AI 마스코트 에셋은 스타일·회전 불일치로 보류(문서 유지, 나중 재검토 여지).
  절차 도형을 생물과 같은 톤 체계로 맞추니 "붕 뜸" 없이 통일되고, 회전도 안전(측면 포즈 이미지의 90° 회전
  어색함 문제가 원천적으로 없음).
- **시각 품질 변경은 스크린샷으로 검증.** 스모크가 "런타임 에러 없음"을 보는 것과 짝으로, `drawBossShape`를
  export 해 프리뷰 하니스가 같은 함수로 그리게 함(화면=검증 1:1). 보스는 이벤트에서만 등장해 인게임 확인이
  어려우니 앞으로 보스 아트를 만질 때 `npm run boss-preview`로 먼저 본다.

## 세션 2 — 보스를 형질에 맞춘 실루엣으로 재설계 (폰 피드백: "다 물고기·바이러스·오징어")

### 요약

③ raider=입 두 개 오징어. 각 보스의 **카운터 형질에 맞춰** 서로 확연히 다른 top-down 실루엣으로 재설계.

### 주요 변경
- `render/worldView.ts` `drawBossShape`:
  - **지느러미 물방울 → 형질별 실루엣.** 육상 포식자엔 top-down **다리**(`leg` 헬퍼: 굵은 라운드 선+발)를
    달아 물고기와 확실히 구분. `fangs`(측면 입) 제거, `frontFangs`(정면 입) 추가.
  - chaser(속도)=질주 육상 맹수(긴 몸·다리 넷 앞뒤로·긴 꼬리·벌린 아가리). swarm(번식)=무는 벌레
    (두 마디 몸·다리 여섯·집게턱). raider(공격력)=**뿔 하나** 들이받는 짐승(코뿔소/투구벌레·다리 여섯,
    입 하나라 오징어 아님). isolation(무리)=배회하는 늑대(뾰족 귀·네 다리·긴 꼬리). stalker(시야)=불규칙
    어둠 덩어리+번뜩이는 눈 셋+뻗는 촉수(대칭 가시공 폐기 → 바이러스 느낌 제거).
  - 공통 스타일(색 파생 굵은 윤곽선·플랫 2톤·큰 흰자 눈)·회전 안전(P 변환)은 세션 1 그대로 유지.

### 결정/기록
- **보스 아트 = 형질 1:1.** 실루엣이 "무엇이 덮치고 뭘 키우면 버티는지"를 형태로 말하게(속도=다리 달린
  질주체, 공격력=뿔, 시야=눈 강조 등). known_issues "시각=로직 1:1"의 연장. 생김새 통일보다 **형질 구분**이
  우선이라는 사용자 방향 반영.
- 프리뷰 하니스(`npm run boss-preview`)로 3방향 회전 재확인 — 5종이 다 다르게, 안 뒤집힘.

## 세션 4 — 시대 보상(큰 시스템 착수: 난이도 루프를 "갈수록 재밌게")

### 요약
사용자가 보스 아트를 보류하고 다른 작업 지시. backlog 큰 시스템 후보 중 **시대 보상** 착수(개체별 진화는
sim 대공사라 폰 검증 없이 자율로 하기 위험 → 더 self-contained 한 이걸로). 시대를 넘을 때마다 강화 카드
3장 중 하나를 고르는 드래프트를 넣어, 난이도가 시대마다 +22% 세지는 걸 성장 도약으로 앞서게 했다.
타입체크·유닛(123, +3)·스모크 통과.

### 주요 변경
- `game/config.ts`: `GAME.eraRewardBoost = 2`(보상 카드 = 정상 ×2 강도).
- `game/cards.ts`: `boostCard(card, boost)` — 효과를 배수로 키운 사본(대가 포함). 표시값(effectiveDelta)과
  실제 적용(applyCard)이 같은 카드 객체를 써 수치 불일치 없음.
- `game/game.ts`: `continueToNextEra`가 곧장 `beginStage` 대신 `beginEraRewardDraft`로 → 시대 보상 드래프트
  (phase=draft, `eraReward` 플래그, 시대 시드 파생 독립 RNG로 결정론). `pickCard`/`skipDraft`에 eraReward
  분기(고르면 갓 태어난 새 시대 무리에 즉시 반영 후 첫 단계로). 언락·비중복 필터는 레벨업과 동일.
- `game/game.test.ts`·`game/cards.test.ts`: continueToNextEra 흐름을 보상 드래프트 경유로 갱신 + boostCard
  강도/표시일치/결정론 테스트 추가.

### 결정/기록
- **개체별 진화 대신 시대 보상**부터. 개체별 진화는 sim 코어 대공사 + 밸런스 재조정이라 폰 없이 자율 완성이
  위험(known_issues: sim 행동 바꾸면 분포 이동). 시대 보상은 game/meta 층 self-contained + 결정론 테스트로
  검증 가능 + 실제 약점("난이도 루프가 어렵기만") 직접 해소.
- **보상 강도는 카드 사본으로.** applyCard 에 boost 인자를 넣는 대신 boostCard 로 값을 키운 사본을 만들어
  드래프트에 넘긴다 → UI(effectiveDelta)와 적용이 같은 소스라 "표시≠실제" 함정(known_issues) 회피.
- **보상은 새 시대 첫 무리에 즉시 반영**(레벨업의 "이후 새끼만"과 달리) — 갓 시작한 무리라 바로 강해지는 게
  보상답다. era 유지·게놈 유지 등 기존 continueToNextEra 불변식은 보존.

### 미해결 / 다음 할 일
- **시대 보상 폰 튠** — 보상 도약(×2)이 난이도 상승을 기분 좋게 앞서는지 폰 체감 후 boost 조정
  (약하면↑, 후반이 시시하면↓/시대별 감쇠). backlog "시대 보상 폰 튠".
- 보스 아트는 사용자와 나중에 재논의(보류).

  플레이어 레벨이 필요. 언락(프리셋·카드·리롤)을 이 레벨 티어로 통합해 진척도 화면이 레벨업마다 해금을
