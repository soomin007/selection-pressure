# 2026-06-25 세션 로그

## 요약
기획서 v0.1 을 바탕으로 "적자생존(Selection Pressure)" 프로젝트를 새로 생성했다.
TypeScript + PixiJS v8 + Vite 스캐폴드를 세우고, 기획서가 "지금부터 챙기라"고 명시한 시드 RNG와
버전 붙은 직렬화 게놈을 구현했다. 오목 프로젝트용으로 받은 CLAUDE.md 를 이 프로젝트에 맞게 다시 썼다.

## 주요 변경
- 빌드 설정: `package.json`, `tsconfig.json`(strict 전체), `vite.config.ts`(`@/`=src 별칭, base `./`)
- `index.html` — 모바일·세로 뷰포트, body flex 중앙정렬(데스크톱 레터박스)
- `src/config.ts` — 논리 해상도 540×960, 색상
- `src/sim/rng.ts` — 시드 고정 결정론 RNG(mulberry32 + xmur3 해시), 상태 저장/복원
- `src/sim/genome.ts` — 버전 붙은(`GENOME_VERSION=1`) 직렬화 게놈, `migrateGenome`/직렬화, `TRAIT_LABELS`
- `src/render/viewport.ts` — scale-to-fit(비율 유지, CSS 크기만 조정)
- `src/main.ts` — PixiJS v8 부트 + 플레이스홀더 무대(틱커·RNG 연결 확인용 표류 점들)
- `src/sim/rng.test.ts` — 결정론 회귀 테스트(4개, 통과)
- 문서: `CLAUDE.md`(전면 재작성), `README.md`, `docs/INDEX.md`, `docs/ROADMAP.md`,
  `docs/design/{backlog,known_issues,open_questions}.md`
- `.gitignore` — node_modules/dist/`ACTIVE_WORK.md` 등

## 결정 사항
- **스택은 기획서대로 TS+PixiJS v8, 빌드는 Vite.** 전역 CLAUDE.md 의 Godot/GDScript 규칙은
  이 프로젝트에 해당 없음 → 프로젝트 CLAUDE.md 에 TS/Pixi 컨벤션을 새로 명시했다.
- **시뮬/렌더 분리를 1순위 구조 규칙으로 못박음.** `src/sim/` 은 Pixi import 금지·순수 TS·결정론.
  기획서의 "Pixi 는 그리기만" 원칙을 코드 경계로 강제.
- **결정론 우선:** `Math.random` 금지, 시드 RNG 주입. 게놈은 버전 붙은 직렬화 구조로 처음부터 설계
  (Phase 1 공짜로 챙기는 §3.1/§6.4 항목).
- 화면 방향은 일단 **세로(540×960)** 로 진행(기획서 §11-1 제안 기본값). 추후 확정 대상으로 남김.
- 스캐폴드는 Phase 1 시뮬을 미리 짓지 않고 토대(RNG·게놈·뷰포트)만 견고히 두었다.

## 세션 3 — Phase 1 시뮬 코어
폰에서 Phase 0 확인됨. 사용자가 "주소창/내비바가 위아래를 먹으니 9:16 이 맞다" 판단 →
해상도 540×960 으로 복귀. 이어 Phase 1 시뮬 코어를 구현했다.

### 주요 변경
- `src/config.ts` — 논리 해상도 540×960(9:16) 복귀
- `src/sim/params.ts` — 시뮬 튜닝 상수 한곳 모음
- `src/sim/entity.ts` · `src/sim/food.ts` — 개체/먹이 데이터 구조
- `src/sim/behavior.ts` — 규칙기반 행동(채집·허기·노화·죽음·번식). 모든 무작위는 world.rng
- `src/sim/world.ts` — World 클래스: 시드로 종 게놈 1개 생성(모든 개체 공유) + 먹이/개체 배치 +
  `step()`(고정 타임스텝). 먹이는 먹히면 일정 틱 뒤 재생(정적 맵 위 지속 생태)
- `src/render/worldView.ts` — 먹이=초록 점, 개체=에너지 밝기. Graphics 재드로우
- `src/render/hud.ts` — 개체 수/먹이/틱, 멸종 안내
- `src/main.ts` — 고정 타임스텝 루프 + 멸종 시 새 시드로 재시작
- `src/sim/world.test.ts` — 결정론(같은 시드→동일 지문) + 생존 sanity(4시드×2000틱 멸종/폭발 없음)

### 결정/기록
- **시뮬/렌더 분리 준수:** sim/ 은 Pixi 미import, 결정론. render 는 sim 을 읽기만.
- **고정 타임스텝(30/s)** 으로 step → 프레임률 무관 결정론. 멸종 재시작은 표시용(테스트는 World 직접).
- **한 런 = 한 종:** 종 게놈 1개를 모든 개체가 공유(참조). Phase 2 에서 카드로 이 게놈을 변형 예정.
- 형질 연결 현황: speed·vision·metabolism·fertility 는 행동에 반영. attack·herding·diet 는 정의만
  하고 미연결(각각 전투·boids·육식 = Phase 1후반/Phase 2).
- 균형: 프로브로 개체 수 추이 확인 → 시드마다 65~103 / ~30 / 70→27 등 종 게놈별로 뚜렷이 갈림.
  이미 "게놈 다르면 결과 다름"이 보임(Phase 2 토대). 폭주(상한 600) 없음.

### 미해결 / 다음
- Phase 2(게놈/형질 의도적 비교) 착수 → `docs/design/backlog.md`.
- 성능: `nearestFood` 완전탐색 → 공간 그리드, 렌더 스프라이트화(개체 수 늘리기 전).

## 세션 4 — Phase 2 게놈/형질 (형질 슬라이더 패널)
사용자가 "①번(슬라이더 패널)으로 쭉 진행" 지시. 형질을 직접 만지며 같은 환경에서 운명이
어떻게 갈리는지 보는 도구를 만들었다.

### 주요 변경
- `src/sim/world.ts` — **환경 시드와 게놈 분리**. `World(seed, w, h, genome)` 로 게놈 외부 주입.
  randomGenome 생성 책임 제거(호출부로 이동). 공유 게놈 객체를 살아있는 중 수정하면 즉시 반영.
- `src/ui/traitPanel.ts` (신규) — HTML 오버레이 슬라이더(속도/시야/대사/번식력) + 버튼
  (같은 환경에서 다시 / 새 환경). 터치 친화(`input[type=range]`).
- `src/main.ts` — 게놈(중립 기본값)·envSeed 분리 보관, 패널 콜백 배선(즉시 적용/재시작/새 환경),
  멸종 시 같은 환경+같은 형질로 재시작(실험 정체성 유지).
- `src/render/hud.ts` — 멸종 문구를 "다시 시작합니다"로 일반화.
- `src/sim/params.ts` — 식량 희소화(90→64) + 개당 가치↑(30→36). 시야/속도가 먹이 선점에
  의미를 갖도록.
- `src/sim/world.test.ts` — 게놈 주입형으로 갱신 + Phase 2 테스트("같은 환경, 형질만 다르면
  결과가 달라진다") 추가. 총 9개 통과.

### 결정/기록
- **환경 시드 ↔ 게놈 분리가 Phase 2의 핵심.** 같은 맵에서 형질만 바꿔야 공정 비교(§3.4).
- 패널은 **탐색/검증 도구**(HTML 오버레이). 실제 게임 UI(카드 3장 드래프트)는 Phase 4.
- 균형 프로브(같은 env, 다른 형질, 2400틱): 대식가 ~26 vs 다산 ~59(2.3배), 둔함/느림 ~33 vs
  예민/빠름 ~56. 네 형질 모두 체감 → "게놈 다르면 결과 다름" 화면에서 확인 가능.
- 식량 밀도가 시야/속도의 가치를 좌우함을 확인 → Phase 3(환경별 유리 형질)의 자연스러운 토대.

### 미해결 / 다음
- Phase 3(절차 환경) 착수 → `docs/design/backlog.md`.
- Phase 2 이월: diet(육식) 행동 연결, 시야 반경 디버그 오버레이, herding(boids).

## 세션 5 — Phase 3 절차 환경
사용자가 "다음 단계 진행" + "슬라이더 영향이 얼마나인지 잘 안 보인다" 피드백. 절차 환경을
넣으면서 가독성 보강(환경 시각화 + 개체 수 추이 그래프)도 함께 했다.

### 주요 변경
- `src/sim/environment.ts` (신규) — 환경 시드로 칸 격자에 coldness/fertility 필드 생성
  (무작위 → 2회 블러 → base 주변 펼침). 맵 전체 한온 성향(coldBase 0.15~0.85)이 맵마다 다름.
- `src/sim/world.ts` — Environment 생성, 먹이를 비옥한 칸에 가중 배치.
- `src/sim/behavior.ts` — 추운 칸에서 저대사 개체에 추가 소모(coldDrain). 수명을 대사와 분리.
- `src/sim/params.ts` — cellSize 60, coldPenalty 0.30, 식량 64/개당 36.
- `src/render/worldView.ts` — 환경 배경(추위 파랑/따뜻 빨강/비옥 밝기) 런 바뀔 때 1회 그림.
- `src/render/hud.ts` — 개체 수 추이 스파크라인(약 23초 창) + reset().
- `src/main.ts` — startWorld()로 월드 교체 시 환경 배경 redraw + 그래프 reset.
- `src/sim/world.test.ts` — Phase 3 테스트(같은 게놈도 맵 따라 결과 갈림) + sanity 에 env-1 추가.

### 결정/기록
- **환경이 유리 형질을 뒤집는다(핵심).** 추운 맵=고대사 유리, 따뜻한 맵=저대사 유리.
  프로브 확인: m4(추위0.17) 저대사46>고대사23 / m5(추위0.78) 고대사21>저대사8.
- **함정→교훈:** 처음엔 고대사가 "기본 소모↑ + 수명↓" 이중 페널티라 추위 저항으로도 못 이겨
  방향이 안 뒤집혔다. 수명을 대사에서 분리(§3.1: 대사=소모/내한성, 수명 언급 없음) + coldPenalty
  0.17→0.30 으로 해결. 깨끗한 트레이드오프가 환경 다양성을 살린다.
- **가독성 보강:** 천천히 변하는 개체 수를 폰에서 잡기 어렵다는 피드백 → 추이 그래프 추가.
  본격 연출(死因·하이라이트)은 Phase 6, 그래프는 그 선행 맛보기.
- env-1(첫 로드)은 따뜻한 맵(추위0.24)이라 기본 게놈이 건강히 생존 → 첫인상 OK.

### 미해결 / 다음
- Phase 4(카드 드래프트 메타루프: 드래프트→관전→정산) 착수 → `docs/design/backlog.md`.
- 슬라이더 패널은 Phase 4 에서 디버그로 격하/토글, 실제 입력은 카드 드래프트로.
