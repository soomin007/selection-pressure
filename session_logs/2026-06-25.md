## 결정 사항
- **스택은 기획서대로 TS+PixiJS v8, 빌드는 Vite.** 전역 CLAUDE.md 의 Godot/GDScript 규칙은
  이 프로젝트에 해당 없음 → 프로젝트 CLAUDE.md 에 TS/Pixi 컨벤션을 새로 명시했다.
- **시뮬/렌더 분리를 1순위 구조 규칙으로 못박음.** `src/sim/` 은 Pixi import 금지·순수 TS·결정론.
  기획서의 "Pixi 는 그리기만" 원칙을 코드 경계로 강제.
- **결정론 우선:** `Math.random` 금지, 시드 RNG 주입. 게놈은 버전 붙은 직렬화 구조로 처음부터 설계
  (Phase 1 공짜로 챙기는 §3.1/§6.4 항목).
- 화면 방향은 일단 **세로(540×960)** 로 진행(기획서 §11-1 제안 기본값). 추후 확정 대상으로 남김.
- 스캐폴드는 Phase 1 시뮬을 미리 짓지 않고 토대(RNG·게놈·뷰포트)만 견고히 두었다.

### 결정/기록
- **시뮬/렌더 분리 준수:** sim/ 은 Pixi 미import, 결정론. render 는 sim 을 읽기만.
- **고정 타임스텝(30/s)** 으로 step → 프레임률 무관 결정론. 멸종 재시작은 표시용(테스트는 World 직접).
- **한 런 = 한 종:** 종 게놈 1개를 모든 개체가 공유(참조). Phase 2 에서 카드로 이 게놈을 변형 예정.
- 형질 연결 현황: speed·vision·metabolism·fertility 는 행동에 반영. attack·herding·diet 는 정의만
  하고 미연결(각각 전투·boids·육식 = Phase 1후반/Phase 2).
- 균형: 프로브로 개체 수 추이 확인 → 시드마다 65~103 / ~30 / 70→27 등 종 게놈별로 뚜렷이 갈림.
  이미 "게놈 다르면 결과 다름"이 보임(Phase 2 토대). 폭주(상한 600) 없음.

- 식량 밀도가 시야/속도의 가치를 좌우함을 확인 → Phase 3(환경별 유리 형질)의 자연스러운 토대.

## 세션 5 — Phase 3 절차 환경
사용자가 "다음 단계 진행" + "슬라이더 영향이 얼마나인지 잘 안 보인다" 피드백. 절차 환경을
넣으면서 가독성 보강(환경 시각화 + 개체 수 추이 그래프)도 함께 했다.

### 주요 변경
- `src/sim/environment.ts` (신규) — 환경 시드로 칸 격자에 coldness/fertility 필드 생성
  (무작위 → 2회 블러 → base 주변 펼침). 맵 전체 한온 성향(coldBase 0.15~0.85)이 맵마다 다름.
- `src/sim/world.ts` — Environment 생성, 먹이를 비옥한 칸에 가중 배치.
- `src/sim/behavior.ts` — 추운 칸에서 저대사 개체에 추가 소모(coldDrain). 수명을 대사와 분리.
- `src/sim/params.ts` — cellSize 60, coldPenalty 0.30, 식량 64/개당 36.
- `src/render/worldView.ts` — 환경 배경(추위 파랑/따뜻 빨강/비옥 밝기) 런 바뀔 때 1회 그림.
- `src/render/hud.ts` — 개체 수 추이 스파크라인(약 23초 창) + reset().
- `src/main.ts` — startWorld()로 월드 교체 시 환경 배경 redraw + 그래프 reset.
- `src/sim/world.test.ts` — Phase 3 테스트(같은 게놈도 맵 따라 결과 갈림) + sanity 에 env-1 추가.

### 결정/기록
- **환경이 유리 형질을 뒤집는다(핵심).** 추운 맵=고대사 유리, 따뜻한 맵=저대사 유리.
  프로브 확인: m4(추위0.17) 저대사46>고대사23 / m5(추위0.78) 고대사21>저대사8.
- **함정→교훈:** 처음엔 고대사가 "기본 소모↑ + 수명↓" 이중 페널티라 추위 저항으로도 못 이겨
  방향이 안 뒤집혔다. 수명을 대사에서 분리(§3.1: 대사=소모/내한성, 수명 언급 없음) + coldPenalty
  0.17→0.30 으로 해결. 깨끗한 트레이드오프가 환경 다양성을 살린다.
- **가독성 보강:** 천천히 변하는 개체 수를 폰에서 잡기 어렵다는 피드백 → 추이 그래프 추가.
  본격 연출(死因·하이라이트)은 Phase 6, 그래프는 그 선행 맛보기.
- env-1(첫 로드)은 따뜻한 맵(추위0.24)이라 기본 게놈이 건강히 생존 → 첫인상 OK.

### 미해결 / 다음
- Phase 4(카드 드래프트 메타루프: 드래프트→관전→정산) 착수 → `docs/design/backlog.md`.
- 슬라이더 패널은 Phase 4 에서 디버그로 격하/토글, 실제 입력은 카드 드래프트로.

### 결정/기록 (밸런스 함정 다수)
- **보스는 "정해진 관문 통과"(버티기)로 구현** — 정밀 창발 전투 밸런싱 회피(기획서 §4).
- **빠른 보스 변별은 좁은 밴드:** 보스 속도가 개체 max(≈1.5~2.3) 근처여야 속도 형질이 갈린다.
  너무 빠르면(4.4) 전원 잡히고, 3.2 부근이 최적(기본15 vs 빠름31).
- **swarm 번식 카운터 실패:** 생태가 먹이 제한이라 번식으로 못 메움 → "건강한 개체 수" 체크로
  완화(솎음률 0.24%/틱).
- **한파 캡 버그:** coldDrain 이 min(1, coldness+globalCold)로 캡 → 따뜻한 맵에선 globalCold 무효.
  캡 제거(대멸종 한파는 별도 가중)로 해결. 한파 22 vs 1 로 변별됨.
- **대가뭄은 완전 중단 금지:** foodFrozen(재생 0)은 전원 아사 → foodRegrowMultiplier(2.5배 느림)로
  바꿔 저대사가 버티게(18 vs 5).

- herding 연결 후에도 Phase 5 밸런스 회귀 테스트(독/한파 필터) 통과 → 기존 균형 유지됨.

## 세션 9 — 식성 모델 재설계 + 형태 그래픽 + 보스 RNG 완화
사용자 피드백: (1) 식성은 시작에 정하고 반대 형질 얻으면 잡식이 맞다, (2) 육식 고르면 즉사,
(3) 점에 색만 말고 형질 기반 형태 그래픽도 미리 고려, (4) 독안개만 여러 번 만나 죽음 = RNG?

### 주요 변경
- 식성 3구간: 초식(<0.35) 식물만 / 잡식(0.35~0.7) 둘 다 / 육식(>0.7) 사냥만.
  잡식이 둘 다 먹어 지속 가능 → 순수 육식 자멸 문제 해결. (`behavior.ts`, `params.dietHuntMin/GrazeMax`)
- **시작 식성 선택**: 런 첫 드래프트가 초식/잡식/육식 카드(set 효과). 육식 시작=diet 0.65(잡식
  영역, 사냥 위주지만 식물도 먹어 생존). (`cards.DIET_CHOICE_CARDS`, `game.firstChoice`)
- **형태 그래픽**: 초식=원, 잡식=마름모, 육식=삼각형(진행 방향), 크기는 공격력.
  `worldView.drawCreature()` 로 분리 — 나중에 스프라이트로 이 함수만 교체.
- **보스 RNG 완화**: 한 런의 두 보스는 서로 다른 종류 강제(shuffle). 독안개 흡수 0.42→0.3,
  추격자 2.9/16 로 완화 → 카운터 없이도 건강한 무리는 버팀(RNG 벽 해소). 보스 통과기준 8→6.
- 대멸종은 클라이맥스 필터로: 통과기준 10, 대가뭄 3.6.

### 결정/기록
- **독안개 죽음 = 부분적으로 RNG + 과한 난도.** 보스가 "정확한 카운터 없으면 즉사"라 카운터
  카드가 안 뜨면 손쓸 수 없었음(기획서가 경고한 RNG 벽). → 보스는 "건강하면 버티되 카운터면
  여유"로 완화, 한 런에 같은 보스 안 나오게. 대멸종을 진짜 필터로.
- 최종 밸런스: 식성(잡식44/육식시작34/순수육식0), 보스(독안개 저대사22·기본10, 추격자 빠름20·
  기본10, 무리 23), 대멸종(한파 고15·저7, 폭염 저36·고4, 대가뭄 저44·고8 / 기준 10).
- drawCreature 분리로 형질 기반 스프라이트 전환 대비(backlog).

### 미해결 / 다음
- 형질 기반 스프라이트 그래픽(backlog).
- 폰 실플레이로 난이도 체감(특히 대멸종 vs 직전 보스 형질 충돌).

## 세션 13 — 생물 이동 자연스럽게 (관성 조향 + 목표 고정)
사용자 피드백: 생물이 제자리에서 드득드득 떨거나, 방향을 마구 바꾸거나, 가다가 갑자기 멈춘다.
"먹이만 보고 달려가기 때문 아니냐" → 정확. 규칙기반 이동을 자연스럽게 고쳤다.

### 원인
- 매 틱 속도 벡터를 목표 방향으로 **순간 재설정**(`e.vx = dir*maxSpeed`) → 관성 0, 홱 꺾임.
- 매 틱 `nearestFood`/`nearestMatching`를 **새로 계산** → 거리가 비슷한 두 먹이 사이/남이 채갈 때
  목표가 A↔B로 진동 → 방향 반전 진동 → 제자리 떨림(="멈춘 것처럼").

### 주요 변경 (전부 sim/, 다른 세션 렌더 영역과 무겹침)
- `src/sim/behavior.ts` — desired velocity 계산 후 현재 속도로 **일부만 보간(관성)**. 도망은 빠르게
  반응(fleeTurn). 목표 **고정(hysteresis)**: 쫓던 먹이/먹잇감이 유효한 동안 유지, 무효일 때만
  재탐색(목표 진동 제거 + nearestFood 호출 빈도↓). cohesion 을 desired 에 합성.
- `src/sim/entity.ts` — 런타임 목표 메모리(targetFood/targetPrey, 직렬화 안 함).
- `src/sim/params.ts` — steerTurn 0.18 / fleeTurn 0.55 / cruiseFactor 0.6 / targetKeepFactor 1.2,
  globalColdLethality 1.7.
- `src/sim/world.test.ts` — 목표 고정 회귀 테스트(유지 >> 교체). 총 30개 통과.

### 결정/기록
- **매끄러운 이동 = 채집 효율↑ → 개체 수 전반 상승.** 그 바람에 한파 대멸종의 절대 통과기준(10)에서
  저대사가 14로 살아남아 필터가 무뎌짐. → 평상시 추위(env.coldness)는 그대로 두고 **대멸종 한파만**
  치사 배율(globalColdLethality)로 더 매섭게 → 필터 복구(한파 hi16/lo6, 독 lo32/base9, 폭염 lo45/hi3).
  평상시 추위·sanity·Phase3 불변(globalCold=0이라 영향 없음).
- 렌더 보간(30step→60fps)은 별도 폴리시(렌더 세션 영역)라 손대지 않음.

### 미해결 / 다음
- 폰에서 이동이 실제로 매끄러운지 체감(사용자).
- (선택) 렌더 위치 보간으로 30→60fps 잔떨림 추가 완화.
