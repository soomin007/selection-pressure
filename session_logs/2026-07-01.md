## 결정 사항
- **스프라이트 이벤트 대신 좌표 + 최근접 탐색.** 생물 스프라이트는 풀 재사용이라 개체와 1:1 이
  아니다(매 프레임 인덱스가 바뀜) → 스프라이트에 이벤트를 달면 잘못된 개체가 잡힌다. 대신 월드
  레이어를 hit-test 에서 빼고(`eventMode="none"`) 탭을 stage 로 통과시켜, `toLocal` 로 월드 좌표를
  구한 뒤 가장 가까운 개체를 직접 찾는다. 카메라/뷰포트 이중 변환도 toLocal 이 한 번에 푼다.
- **탭 가드 두 겹.** `e.target===app.stage`(범례 등 누를 수 있는 UI 가 잡은 탭 제외) + 미니맵 영역
  제외. 빈 월드/생물 위 탭만 선택으로 처리.
- **카메라는 sim 위치가 아니라 렌더 표시 좌표(dispPos)를 따라간다.** 줌 2.6 클로즈업에서 sim 의
  고주파 떨림이 그대로 보이면 멀미 → worldView 의 저역통과 좌표를 따라가 떨림 없이 추적. 강조 고리도
  같은 좌표라 개체와 정확히 겹친다.
- **개체는 종 게놈을 공유**하므로 형질은 같은 종이면 동일. 그래도 카드에 형질을 보여주는 건 야생
  개체를 탭했을 때 새 정보이고(내 종은 빌드 패널과 같음), "이 아이가 뭘 가졌나"의 즉시 확인용.
  개체별 차이(이름·기운·나이·하는 일)가 애착의 핵심 정보.
- **죽음 = 작별.** 따라가던 개체가 사라지면 "○○ 떠남"을 한 번 띄우고 선택 해제 → 무리로 복귀.
  소수 개체 게임에서 한 마리의 죽음이 무게를 갖게 하는 연출.

## 세션 2 — 스포어식 3종 세트(미니맵 팬 + 개체 개성 + 우호 친척)

### 요약
사용자: "미니맵 꾹 눌러 드래그로 카메라 이동 + 우호적 친척 무리(스포어식) + 다음 단계도". 셋을
A(미니맵 팬) → C(개체 개성) → B(우호 친척) 순으로 각각 커밋. B 는 밸런스 민감이라 마지막에 신중히.
76 테스트 통과.

### 주요 변경
- **A 미니맵 드래그 카메라** (a62a54c): `minimap.ts` interactive + onPan 콜백(포인터→월드 좌표,
  드래그 추적). `main.ts` manualCam 수동 팬(우선순위 개체>수동>보스>무리), 화면 탭 시 해제.
- **C 개체별 미세 개성** (8283722): `creatureLook.ts`(신규) id 결정론 personalityScale(±18%)·
  personalityTint(명암)·sizeWord. `worldView.ts` 스프라이트에 개체별 크기·틴트(sim 무관). 카드에 덩치.
- **B 우호적 친척 무리** (f3a3f7a): `species.ts` friendly 플래그 + makeKinSpecies(온건 잡식 원형에서
  갈라진 종) + areFriends. `behavior.ts` 사냥감·포식자 감지에서 우호 종 제외. `world.ts` 친척 종
  추가(게놈·스폰 독립 rng) + 독립 영역 스폰 + 이주 제외. `params.ts` kinInitialCount 10.

### 결정/기록
- **친척은 내 종 게놈을 물려받지 않는다(중요).** 처음엔 playerGenome 복제로 만들었더니, 밸런스
  테스트가 극단 게놈(vision 0.9 등)으로 World 를 직접 생성 → 친척도 vision 0.9 를 물려받아 똑같이
  강한 경쟁자가 되어 내 종을 굶겼다(hi 케이스 통과 실패). → 온건한 균형 원형에서 미세 흔들림으로
  변경. 설계상으로도 "갈라진 친척은 제 갈 길"이 더 스포어답고, 내 종이 어떻게 진화하든 친척은 독립.
- **친척은 내 종 근처가 아니라 독립 영역에 스폰(중요).** 근처에 두니 소수 개체 + 통과기준 3(빡빡한
  마진)에서 국소 먹이를 함께 소진해 내 종 통과 개체수가 3 미만으로 떨어졌다(외톨이 1·매복자 0·한파 2).
  독립 영역으로 바꾸니 3케이스 전부 회복. 만나면 우호(friendly)로 섞이되 국소 직접 경쟁은 없앤다.
- **밸런스 보존 = 독립 rng(반복 패턴).** 친척 게놈·스폰을 독립 rng(seed-kin/-kinpos)로 만들어 메인
  스트림을 1비트도 안 건드림 → 기존 보스/대멸종 통과기준 무재튠. 지형·바다먹이·야생진화와 같은 전략.
  단 "독립 rng"는 소비 순서(결정론)만 보존 — 친척이 생태에 실재하므로 동역학(먹이 경쟁)은 위 두 결정
  (온건 게놈·독립 영역)으로 눌렀다.
- **친척도 이주 없음 = 내 편(멸종하면 사라짐).** 야생은 이주로 보충되지만 친척은 플레이어처럼 이주
  없다 — 애착 대상이라 죽으면 정말 사라진다. 야생 진화(maybeEvolveWild)는 적용(스포어식 함께 진화).
- **개체 개성은 렌더 전용.** 크기·명암을 sim 에 넣으면 밸런스가 흔들리니 worldView 에서만 id 결정론
  변주. "즉각 시각 피드백"(폰) + 밸런스 무영향 둘 다 만족. 카드의 "덩치"도 같은 함수(sizeWord).

- **폰 검증**: ① 미니맵 꾹 눌러/끌어 카메라가 그 위치로 가는지(빈 곳 탭으로 무리 복귀) ② 개체마다

### 요약
폰 피드백: "친척은 보이는데 색 구분이 잘 안 됨. 미니맵 카메라는 잘 됨. 카드에 이전/다음 화살표를
넣어 내 종의 다른 애로 포커스 옮기게. 다른 작업도 함께". 화살표 + 우호 무리 시각 구분 두 개 진행.
sim 미변경(렌더·UI) → 76 테스트 그대로 통과.

### 주요 변경
- **카드 이전/다음 화살표** (3ef5ce2 앞 커밋): `creatureCard.ts` 헤더에 ‹ › 버튼(콜백을 객체로
  onClose/onPrev/onNext). `main.ts` cycleSelection — 현재 선택 개체와 같은 종 무리 안에서 id 순환
  (수동 조망 해제). 한 마리씩 둘러보는 관찰.
- **우호 무리 시각 구분** (3ef5ce2): `worldView.ts` 친척(friendly)에 청록 고리(내 종 초록 고리와

### 결정/기록
- **화살표는 "같은 종 무리" 순환.** 사용자는 "내 종의 다른 애"라 했지만, 현재 선택 개체의 종으로
  필터해 순환하면 친척·야생을 골랐을 때도 그 무리 안을 둘러볼 수 있어 더 일반적(내 종 의도 포함).
  id 오름차순 순환이라 다음/이전이 직관적. 개체가 죽어 목록에서 빠지면 그 자리에서 멈춘다.
- **색 구분은 "소속 고리"로 푼다(팔레트 재설계 대신).** 종이 10개라 스프라이트 색만으론 구분이 어려운
  건 알려진 한계(backlog). 이번 문제의 핵심은 "어느 게 내 편인가"라, 내 종=초록 고리·친척=청록 고리로
  소속을 테두리에 실었다. 화면 고리와 범례 테두리를 같은 색으로 맞춰 연결된다. 야생은 고리 없음 →
  내 편 두 무리 / 야생이 한눈에 갈린다. 전체 색 가독성(야생 7종)은 별도 과제로 남김.

### 미해결 / 다음 할 일

## 세션 5 — 야생종끼리 친척·동맹(faction)

### 요약
사용자 제안: "야생종끼리도 친척을 만들면 (내 종 밸런스 안 건드리고) 되지 않나?" → 채택. faction(편)
개념 도입 — 같은 편은 서로 사냥·도망 안 함. 내 종+친척=1편, 야생 초원 연합=2편. 근처 동거와 달리
스폰 위치를 안 건드려(사냥 관계만) 내 종 무리·통과기준에 직접 영향 없음. 한파 한 지점만 튠, 77 통과.

### 주요 변경 (4b7d598)
- `species.ts`: Species.faction(편, 0=중립). areFriends 를 `a.faction!==0 && a.faction===b.faction`
  으로 — 내 종↔친척, 야생 동맹끼리를 하나의 판정으로 통합. 초원 연합(초식경쟁자·들풀무리·잡식청소부)
  = faction 2. friendly 플래그는 렌더(내 친척 청록 고리)용으로 유지.
- `worldView.ts`/`hud.ts`: 야생 동맹에 옅은 금빛 고리·범례 테두리(내 편 초록/청록과 구분되는 편 색).
- `params.ts`: globalColdLethality 1.5→1.4(한파 전용).
- `world.test.ts`: 야생 동맹 우호 검증(같은 편 우호·중립끼리 아님).

### 결정/기록
- **behavior 는 한 줄도 안 바뀐다.** 사냥/도망 제외가 이미 areFriends 호출이라, areFriends 내부만
  faction 기반으로 바꾸니 야생 동맹까지 자동 확장. 우호 판정을 한곳(areFriends)에 모아둔 설계의 이득.
- **야생끼리 동맹은 근처 동거보다 근본적으로 안전.** 밸런스가 깨졌던 원인(친척이 내 종 무리 cohesion
  교란)이 야생끼리엔 없다. 스폰 위치도 안 건드려(사냥 관계만) 내 종 통과기준에 직접 영향 없음. 유일한
  간접 영향은 "잡식 청소부가 동맹 초식을 안 잡음→초식 번성→내 종 먹이 경쟁"인데, 한파 통과기준(소수라
  경계)만 살짝 흔들어 한파 전용 지점 하나로 흡수(고대사 2→통과). known_issues 규칙대로 영향 지점만 타깃.
- **동맹에 사냥 성향 종(청소부)을 넣어야 행동이 드러난다.** 초식끼리만 묶으면 원래 안 싸워 동맹이
  시각뿐. 청소부(잡식·사냥함)가 동맹 초식을 안 잡는 게 관찰 포인트("저 종은 청소부랑 한 편이라 안 먹히네").
