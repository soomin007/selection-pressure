## 세션 2 — 개체별 개성(무늬·눈·비등방 크기·색조) (다음 세션 첫 작업 이행)

### 요약
같은 종 안에서 개체가 안 달라 보이던 문제(전엔 크기 ±18%·회색 명암만 = 미묘)를 두 층으로 해결.
① 텍스처 변형: 개체 id 를 유한한 "룩 버킷"(16종)으로 해시해 무늬(줄무늬/반점/얼룩/민무늬)·눈 크기/위치가
다른 텍스처를 만든다. ② 스프라이트 변주: 비등방 크기(길쭉/통통)·명암+따뜻/차가운 색조를 개체마다 연속 부여.
순수 렌더(밸런스·결정론 무관). 타입체크·빌드 통과.

### 주요 변경
- `render/creatureLook.ts`: `CreatureLook`(무늬 종류·반점 배치·눈 변형) + `LOOK_BUCKETS`=16 버킷,
  `lookBucket`/`lookFromBucket`/`creatureLook`(id→룩). `personalityStretch`(길쭉함) 추가. `personalityTint`
  를 회색 명암 → 명암+따뜻/차가운 톤으로. `DEFAULT_LOOK`(무늬 없는 종 대표 — 프리셋용).
- `render/worldView.ts`: `makeCreatureTexture`에 `look` 파라미터 optional(기본 DEFAULT_LOOK →
  presetPanel 호출부 무변경). `drawPattern` 헬퍼(줄무늬=세로 줄, 반점/얼룩=등쪽 타원). 눈이 look 반영.
  `textureFor` 캐시 키에 룩 버킷 추가(캐시 = 게놈 서명 × 버킷, 유한 상한). sprite scale 비등방(ps·st, ps/st).

### 결정/기록
- **텍스처 변형은 유한 버킷, 스프라이트 변주는 연속** — 무늬·눈은 텍스처라 개체마다 새로 만들면 캐시 폭증.
  그래서 id→16버킷으로 이산화(종당 텍스처 상한 유지). 크기·색조는 스프라이트 속성이라 공짜 연속 변주.
  두 층이 겹쳐 버킷이 같은 두 마리도 크기·톤으로 갈린다.
- **무늬 좌표는 몸 반길이 len·반너비 wid 안쪽으로** — 윤곽선을 안 넘게. 기존 배 그림자/눈과 같은
  ellipse/stroke API라 저위험. 헤드리스 렌더 불가라 배치는 폰 검토로 최종 확인.

### 폰 검증 대기
- 무늬(줄/반점/얼룩)가 몸에 잘 얹히고 종별 실루엣을 안 해치는지. 같은 종 무리가 이제 한 마리씩 달라
  보이는지(애착 목적 달성). 무늬 대비가 과하면 darken/lighten 계수↓, 약하면↑. 눈 변형이 어색하면 범위↓.

### 결정/기록
- **era 0 = 완전 무영향이 안전의 핵심** — 난이도 배율을 곱셈(1+era*step)으로 해 첫 시대는 곱이 1. 그래서

  brotato 처럼 "생존 조건은 같고 적이 세진다". 성장(레벨·형질) vs 난이도 상승의 경주.
- **다음 시대 = 새 맵·초기 무리 리셋, 게놈·레벨은 유지** — 성장한 종으로 더 험한 새 땅에 다시 시작. 시드는
  `baseSeed-eraN` 으로 파생해 결정론 유지(시대마다 다른 판).
- **프리셋 극단화는 preset_omni 제외** — game.test 가 omni 로 6단계 완주를 검증하므로 그것만 보존.

## 세션 5 — 보스 그래픽 개성 + 버튼 강조 + 원거리 임계화 (폰 피드백 묶음)

### 요약
폰 피드백 6건. 실행 3 + 답변 3. 실행: ① 보스 6종 종류별 실루엣(다 비슷한 점 → 형태로 구분) ② "다음
시대로" 버튼 강조·분리(붙어있고 색 비슷 → 크고 밝은 주 버튼 + 은은한 보조) ③ 원거리를 임계 기반으로
바꿔 전문종만 확실히 멀리서 쏨(밸런스 불변). 타입체크·빌드·테스트(111) 통과.

### 주요 변경
- `render/worldView.ts`: `drawPredatorDot` → `drawBossCreature`(진행 방향 회전 + 종류별 실루엣) — chaser=
  화살촉·swarm=삼각·raider=화살촉쐐기·isolation=칼날마름모·stalker=그림자몸+번뜩이는 눈. HORDE_COLORS 에
  chaser 색 추가, BOSS_OUTLINE 상수. 물기 반경·맥동 고리는 유지(게임성·로직 1:1).

  작은 테두리 보조 버튼으로 분리(여백↑). 패배 시엔 새 런이 파랑 주 버튼.

  기존 기울기)·`rangedBonusHigh 130`(임계 초과 급기울기). atkRange = 근접 + 저구간 + 고구간.

### 결정/기록
- **원거리는 임계 기반 능력으로** — rangedBonus 를 그냥 22→62 로 올리니 야생/부수적 ranged 사거리도 늘어
  knife-edge swarm 통과기준 테스트가 깨졌다(known_issues: param 올리면 야생 생태 이동). 해결: 임계 이하
  구간은 **기존 기울기 그대로**(ranged≤55 는 사거리가 byte-identical → 모든 테스트 보존) + 초과분만 급기울기.
  전문 원거리 종(프리셋 82 → 사거리 ≈59px)만 확 멀어진다. 사용자의 "3단계 형질" 제안과도 부합(none/minor/전문).
- **보스 실루엣은 진행 방향으로 회전** — 헤딩(x-prevX) 으로 로컬→월드 회전. 거의 안 움직이면 아래(+y)
  향함(몰려오는 방향). 물기 반경·맥동은 유지(시각=로직 1:1, known_issues).

### 답변만(실행 안 함 — 사용자 판단 대기)
- **초음파**: echoBase 130 < visionBase 200 이라 밝은 낮·트인 곳에선 시야가 더 넓어 초음파 이점이 안
  드러난다(진가는 밤·수풀). 프리셋도 없어 경험 접점 적음. → 개선안: 초음파 프리셋 추가 or echoBase↑ or
  시각에 "감지 범위" 설명. 사용자 선택 대기.
- **속도**: 범용 형질(채집·도망·사냥·지형 다 유리)이라 체감상 제일 유리한 게 맞다. 단 시련은 각각 다른
  카운터 형질(고/저대사·공격·무리·시야)을 요구해 속도만으론 못 이김. 조정하려면 속도 범용 이점↓ or 다른
  형질 상황↑. 사용자 판단 대기.
- **3단계 형질**: 좋은 지적. wings/swimming 은 이미 임계 기반, ranged 도 이번에 임계화. echo/venom 도 같은
  임계화 + UI 3단계 표시(없음/보통/강함)가 직관적. 표시 변경은 traitPanel/presetPanel/glossary 범위 → 원하면 착수.

### 폰 검증 대기
- 보스 5종 실루엣이 종류별로 구분되고 진행 방향으로 잘 향하는지. 크기(떼 8.5·단일 15) 적절한지.
